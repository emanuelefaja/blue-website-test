查找自定义字段允许您从其他项目的记录中导入数据，创建单向可见性。虽然[引用自定义字段](/zh/docs/custom-fields/reference)建立跨项目连接，但查找字段使用这些连接来显示选定的数据。

查找自定义字段始终是只读的，并连接到特定的引用自定义字段。当用户使用引用自定义字段选择一个或多个记录时，查找自定义字段将显示来自这些记录的数据。

查找字段可以显示以下数据：

- **创建时间**：引用记录的创建时间。
- **更新时间**：引用记录最后更新的日期和时间。
- **截止日期**：引用记录的开始/截止日期和时间。
- **描述**：引用记录的描述字段。
- **列表**：显示引用记录所属的列表。
- **标签**：引用记录的任何标签。
- **指派人员**：引用记录的任何指派人员。
- **任何自定义字段**：引用记录中任何支持的自定义字段。

## 查找的查找

您可以创建自定义字段来存储和引用跨多个不同项目的数据。这些自定义字段称为"查找"字段。

让我们考虑三个项目：

- **项目 A**
- **项目 B**
- **项目 C**

假设项目 B 包含一些与项目 A 和项目 C 都相关的关键信息或资源。

与其将项目 B 的信息复制到项目 A 和项目 C 中，不如按照以下步骤操作：

- 在项目 A 中，创建一个"引用"自定义字段，以与项目 B 中的相关记录建立连接。

  - 此引用自定义字段允许您选择并链接到项目 B 中您想要引用和查找的特定记录。

- 在项目 A 中设置引用字段后，在项目 C 中创建一个"查找"自定义字段。

  - 此查找自定义字段将连接到项目 A 中的引用自定义字段。

  - 当您在项目 A 中使用引用自定义字段选择记录时，项目 C 中的查找自定义字段将自动显示来自项目 B 中这些引用记录的相关数据。

## 使用查找的查找的好处：

- 您无需直接将项目 C 链接到项目 B。
- 项目 C 可以通过引用和查找字段利用项目 A 和项目 B 之间建立的连接。
- 对项目 B 中相关记录所做的任何更改或更新将自动反映在项目 C 中显示的查找自定义字段数据中。
- 无需在项目之间手动更新或复制信息。
