El campo ID Único genera automáticamente un identificador único para cada registro. Blue ofrece dos tipos de sistemas de identificación únicos:

## 1. IDs Alfanuméricos Aleatorios (Por Defecto)

El sistema estándar de ID Único genera un identificador alfanumérico único para cada registro. Esto aporta un valor inmenso al crear claves persistentes que identifican consistentemente cada registro. Incluso si un registro se elimina, su ID nunca se duplicará.

## 2. IDs Secuenciales

Para proyectos que requieren identificadores numerados legibles por humanos, Blue también soporta IDs secuenciales. Cuando se habilita, este sistema genera números incrementales (ej. 1, 2, 3, o con relleno de ceros como 001, 002, 003) que pueden personalizarse con prefijos (ej. TAREA-001, TICKET-002).

## Casos de Uso

Ambos sistemas de ID habilitan capacidades poderosas:

**Beneficios Generales:**
- Asociar sistemas externos a registros mediante códigos únicos
- Crear registros de auditoría rastreando referencias de ID
- Buscar registros fácilmente en la plataforma mediante el ID, incluso si el nombre del registro cambia

**Beneficios Específicos de ID Secuencial:**
- Referencias de tareas legibles por humanos para comunicación del equipo
- Esquemas de numeración consistentes para flujos de trabajo (ej. tickets de soporte, solicitudes de características)
- Integración fácil con sistemas externos que esperan identificadores numéricos
- Generación automática de títulos con prefijos secuenciales

Ambos sistemas de ID establecen esencialmente "números de seguridad social" inmutables para cada registro. Esta permanencia y especificidad habilita integraciones poderosas, análisis y capacidades de identificación.

## Configuración de ID Secuencial

Al crear o editar un campo personalizado de ID Único, puede habilitar la numeración secuencial con estas opciones:

- **Usar ID Único Secuencial**: Alternar para habilitar numeración secuencial en lugar de IDs alfanuméricos aleatorios
- **Dígitos de Secuencia**: Número de dígitos para el relleno de ceros (ej. 3 dígitos = 001, 002, 003)
- **Número de Inicio de Secuencia**: El número desde el cual comenzar a contar (por defecto: 1)
- **Prefijo**: Prefijo de texto opcional para títulos generados (ej. "TAREA-" crea TAREA-001, TAREA-002)

**Total de IDs Únicos Disponibles** <br/><br/>
Hay 36 posibilidades para cada uno de los 25 caracteres en un ID Único de Blue: <br/>
- 26 letras mayúsculas (A-Z) <br/>
- 10 dígitos (0-9) <br/>

Esto significa que hay un total de 8.08×10^35 posibilidades o, para ser más precisos: 808,281,277,464,764,060,643,139,600,456,536,293,376 <br/><br/>
Para poner esto en comparación, ¡por cada grano de arena en la Tierra (7.5*10^18), hay decenas de cuatrillones (1.08×10&17) de IDs Únicos!

**Detalles Técnicos de ID Secuencial** <br/><br/>
Los IDs secuenciales tienen alcance de proyecto, lo que significa que cada proyecto mantiene su propio contador de secuencia. Cuando los registros se copian o mueven entre proyectos, el sistema maneja inteligentemente la asignación de secuencias basada en configuraciones de campos personalizados que coinciden. La generación de secuencias es atómica y segura para hilos para prevenir números duplicados en entornos concurrentes.
