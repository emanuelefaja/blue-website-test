Los campos personalizados de búsqueda le permiten importar datos de registros en otros proyectos, creando visibilidad unidireccional. Mientras que los [Campos personalizados de referencia](/es/docs/custom-fields/reference) establecen conexiones entre proyectos, las búsquedas utilizan esas conexiones para mostrar datos seleccionados.

Los campos personalizados de búsqueda son siempre de solo lectura y están conectados a un campo personalizado de referencia específico. Cuando un usuario selecciona uno o más registros usando un campo personalizado de referencia, el campo personalizado de búsqueda mostrará datos de esos registros.

Las búsquedas pueden mostrar datos como:

- **Creado en**: Cuándo se creó el registro referenciado.
- **Actualizado en**: La última fecha y hora cuando el registro referenciado se actualizó de alguna manera.
- **Fecha de vencimiento**: La fecha y hora de inicio/vencimiento del registro de referencia.
- **Descripción**: El campo de descripción del registro referenciado.
- **Lista**: Muestra la lista a la que pertenece el registro referenciado.
- **Etiqueta**: Cualquier etiqueta con la que esté etiquetado el registro referenciado.
- **Asignado**: Cualquier asignado al registro referenciado.
- **Cualquier campo personalizado**: Cualquier campo personalizado compatible del registro referenciado.

## Búsqueda de una búsqueda

Puede crear campos personalizados para almacenar y referenciar datos a través de múltiples proyectos diferentes. Estos campos personalizados se llaman campos de "Búsqueda".

Consideremos tres proyectos:

- **Proyecto A**
- **Proyecto B**
- **Proyecto C**

Supongamos que el Proyecto B contiene información crítica o recursos que son relevantes tanto para el Proyecto A como para el Proyecto C.

En lugar de duplicar la información del Proyecto B en el Proyecto A y el Proyecto C, siga estos pasos:

- En el Proyecto A, cree un campo personalizado de "Referencia" para establecer una conexión con el registro relevante del Proyecto B.

  - Este campo personalizado de referencia le permite seleccionar y vincular al registro específico en el Proyecto B que desea referenciar y buscar.

- Una vez que el campo de referencia esté configurado en el Proyecto A, cree un campo personalizado de "Búsqueda" en el Proyecto C.

  - Este campo personalizado de búsqueda estará conectado al campo personalizado de referencia en el Proyecto A.

  - Cuando seleccione registros usando el campo personalizado de referencia en el Proyecto A, el campo personalizado de búsqueda en el Proyecto C mostrará automáticamente datos relevantes de esos registros referenciados en el Proyecto B.

## Beneficios de usar Búsqueda de una búsqueda:

- No necesita vincular directamente el Proyecto C al Proyecto B.
- El Proyecto C puede aprovechar la conexión establecida entre el Proyecto A y el Proyecto B a través de los campos de referencia y búsqueda.
- Cualquier cambio o actualización realizada a los registros relevantes en el Proyecto B se reflejará automáticamente en los datos del campo personalizado de búsqueda mostrados en el Proyecto C.
- No hay necesidad de actualizaciones manuales o duplicación de información entre proyectos.
