## Resumen

Las imágenes de portada le permiten agregar contexto visual a sus registros mostrando vistas previas de imágenes directamente en las tarjetas de registro. Esta función extrae automáticamente imágenes de varias fuentes dentro de sus registros y las muestra como miniaturas, facilitando la identificación y organización del contenido visual de un vistazo.

Ya sea que esté gestionando proyectos de diseño, calendarios de contenido, catálogos de productos o cualquier flujo de trabajo que involucre activos visuales, las imágenes de portada le ayudan a revisar rápidamente los registros y encontrar lo que busca. La función incluye seguimiento de progreso en tiempo real y opciones de visualización flexibles para adaptarse a sus necesidades específicas de flujo de trabajo.

## Características Clave

- **Extracción Automática de Imágenes**: Blue encuentra y muestra automáticamente imágenes de su fuente elegida
- **Múltiples Fuentes de Imágenes**: Obtenga imágenes de descripciones de registros, comentarios o campos de archivo personalizados
- **Opciones de Ajuste de Visualización**: Elija cómo aparecen las imágenes con Cubrir o Contener.
- **Selección Inteligente**: Configure si usar la primera o última imagen de su fuente
- **Actualizaciones en Tiempo Real**: Las imágenes de portada se actualizan automáticamente cuando cambia el contenido del registro
- **Seguimiento de Progreso en Vivo**: Monitoree el procesamiento masivo con suscripciones de progreso en tiempo real

## Cómo Funciona

### Habilitando Imágenes de Portada

Las imágenes de portada se configuran a nivel de proyecto por los Administradores de Proyecto:

1. Abra su proyecto y navegue a **Configuración del Proyecto**
2. Seleccione **Configuración de Registros**
3. Encuentre la sección **Opciones de Imagen de Portada**
4. Active **Habilitar Imágenes de Portada** para activar la función

### Opciones de Configuración

#### Fuente de Imagen
Elija dónde debe buscar imágenes Blue:
- **Descripción**: Extrae imágenes incrustadas en las descripciones de registros (predeterminado)
- **Comentarios**: Obtiene la primera/última imagen de los comentarios del registro
- **Campo Personalizado**: Usa archivos de un campo personalizado específico de tipo archivo (admite cualquier tipo de archivo)

#### Selección de Imagen
Cuando hay múltiples imágenes disponibles, elija cuál mostrar:
- **Primera Imagen**: Usa la primera imagen encontrada (predeterminado)
- **Última Imagen**: Usa la imagen más reciente

#### Ajuste de Visualización
Controle cómo aparecen las imágenes en el área de vista previa:
- **Cubrir**: La imagen llena toda el área de vista previa, puede recortar bordes para mantener la relación de aspecto (predeterminado)
- **Contener**: Muestra la imagen completa dentro del área de vista previa, puede tener espacio vacío

## Casos de Uso

**Proyectos de Diseño** - Muestre maquetas e iteraciones de diseño directamente en las tarjetas de tareas, facilitando ver el estado actual de cada tarea de diseño sin abrir registros individuales.

**Planificación de Contenido** - Para calendarios de contenido y flujos de trabajo editoriales, muestre imágenes destacadas o miniaturas para visualizar su pipeline de contenido de un vistazo.

**Gestión de Productos** - Muestre fotos de productos en registros de inventario o solicitudes de funciones, ayudando a los equipos a identificar rápidamente elementos durante las discusiones.

**Bienes Raíces** - Muestre fotos de propiedades en registros de listados, permitiendo a los agentes navegar rápidamente por las propiedades visualmente.

**Planificación de Eventos** - Muestre fotos de lugares, conceptos de diseño o diseños de eventos en tareas de planificación para una mejor organización visual.

## Configurando Imágenes de Portada

### Paso 1: Elija Su Fuente de Imagen

Considere dónde existen naturalmente las imágenes en su flujo de trabajo:
- Si las imágenes son parte de descripciones de tareas o resúmenes, use **Descripción**
- Si las imágenes se agregan a través de colaboración en equipo, use **Comentarios**
- Si tiene un campo dedicado para imágenes, use **Campo Personalizado**

### Paso 2: Configure los Ajustes de Visualización

1. Seleccione su **Ajuste de Imagen** preferido basado en sus tipos de imagen:
   - Use **Cubrir** para fotos y diseños donde el recorte es aceptable
   - Use **Contener** para diagramas o capturas de pantalla donde necesita ver todo

2. Elija **Selección de Imagen** basado en su flujo de trabajo:
   - Use **Primera** para flujos de trabajo donde la imagen principal se agrega inicialmente
   - Use **Última** para flujos de trabajo donde la imagen más reciente es más relevante

### Paso 3: Procese Registros Existentes

Cuando habilita por primera vez las imágenes de portada o cambia la configuración, Blue procesa automáticamente sus registros:
1. Las actualizaciones de progreso en tiempo real muestran el estado del procesamiento
2. Los registros se procesan en lotes (50 registros por lote) para un rendimiento óptimo
3. Puede continuar trabajando mientras el procesamiento ocurre en segundo plano
4. Las listas se actualizarán automáticamente cuando se complete el procesamiento
5. El procesamiento se activa automáticamente cuando se guardan los cambios de configuración

## Detalles Técnicos

### Formatos Compatibles
- Al usar fuentes de Descripción o Comentarios:
  - Imágenes JPEG/JPG
  - Imágenes PNG (incluyendo transparencia)
  - Imágenes GIF (visualización estática)
- Al usar fuente de Campo Personalizado:
  - Cualquier tipo de archivo puede mostrarse (imágenes, PDFs, documentos)
  - Los archivos que no son imágenes muestran iconos apropiados del tipo de archivo
- Todos los archivos deben subirse a Blue (URLs externas no compatibles)

### Limitaciones
- Máximo 6,000 registros por proyecto para generación masiva de portadas
- Solo los campos personalizados de tipo archivo pueden usarse como fuentes
- Las imágenes de portada son solo de visualización y no pueden editarse directamente

### Actualizaciones Automáticas
Las portadas se regeneran automáticamente cuando:
- Se edita la descripción del registro (si usa fuente Descripción)
- Se agregan nuevos comentarios con imágenes (si usa fuente Comentarios)
- Se actualizan archivos en campos personalizados (si usa fuente Campo Personalizado)
- Se cambian las configuraciones de portada
- Se modifica la opción de ajuste de visualización

### Acceso a API
Las imágenes de portada están completamente integradas con la API GraphQL:
- Configure via mutación `setTodoCoverConfig`
- Acceda a datos de portada a través del campo `cover` en el tipo Todo
- Monitoree el progreso con la suscripción `subscribeToCoverGenerationProgress`
- URLs de miniatura generadas automáticamente en `/uploads/{uid}/thumbnail/cover`


## Integración con Otras Funciones

Las imágenes de portada funcionan perfectamente con otras funciones de Blue:
- **[Formularios](/es/docs/projects/forms)**: Las imágenes enviadas pueden convertirse automáticamente en portadas
- **[API](/es/docs/integrations/api)**: Soporte completo de API GraphQL para configuración y acceso a datos
- **[Campos Personalizados](/es/docs/projects/custom-fields)**: Cualquier campo de tipo archivo puede servir como fuente de portada
