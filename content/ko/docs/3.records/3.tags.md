태그는 작업을 정리하는 유용한 방법입니다. 태그를 사용하여 다양한 작업 유형, 긴급도 또는 프로젝트에 적합한 모든 카테고리를 표시할 수 있습니다. 각 태그는 쉬운 식별을 위해 색상 코딩이 가능하며 작업 공간에 색상을 추가할 수 있습니다.

<youtube url="https://www.youtube.com/watch?v=nlyY3cUUpkQ" />


## 태그 사용하기

- **태그 추가**: 태그를 생성하려면 분류가 필요한 작업의 측면을 고려하세요. 'bug', 'back-end', 'UI'와 같은 작업의 성격이나 우선순위 등이 될 수 있습니다. 카테고리를 식별한 후에는 쉽게 태그를 생성하고 즉시 인식할 수 있도록 고유한 색상을 할당할 수 있습니다.
- **태그로 필터링**: Blue의 필터링 기능은 작업량 관리를 더욱 직관적으로 만들어 줍니다. 드롭다운 필터 아이콘을 사용하거나 보드 뷰에서 원하는 태그를 클릭하여 직접 태그를 필터링할 수 있습니다.

프로젝트 간 태그 지정 <br/>
대시보드 및 맵과 같은 조직 전체 뷰의 경우, Blue가 자동으로 같은 이름의 태그를 병합하기 때문에 태그 지정이 특히 중요합니다. 따라서 "Priority 1" 태그로 필터링하기만 하면 해당 태그가 있는 모든 프로젝트의 레코드만 표시됩니다.
