---
title: 프로젝트 아카이브
description: 규정 준수를 위해 완료된 프로젝트를 동결합니다.
---

많은 조직에서 완료된 프로젝트의 데이터를 보존하는 것은 감사 목적이나 향후 프로젝트를 다시 검토하기 위해 중요합니다. 그러나 수백 개의 완료된 프로젝트가 누적되면 Blue 인터페이스가 복잡해져 현재 프로젝트 관리와 탐색이 어려워질 수 있습니다.

이것이 Blue가 [프로젝트 관리자](/docs/user-management/roles/project-administrator)에게 프로젝트가 완료되거나 더 이상 필요하지 않을 때 아카이브할 수 있는 기능을 제공하는 이유입니다.

![Archiving Project 1](/docs/ArchivingProject_1.png "Archiving Project 1")

모든 프로젝트를 쉽게 아카이브

프로젝트가 아카이브되면 여러 가지 일이 발생합니다.

- **'아카이브된 프로젝트'로 이동**: 프로젝트가 사이드바의 메인 '프로젝트' 영역에서 지정된 '아카이브된 프로젝트' 섹션으로 이동됩니다. 이를 통해 깨끗하고 정리된 메인 인터페이스를 유지할 수 있습니다.
- **아카이브 상태 표시**: 상단 바에서 프로젝트 이름 옆에 '아카이브됨' 라벨이 나타나 상태를 명확하게 표시합니다.
- **읽기 전용 액세스**: 아카이브된 프로젝트는 읽기 전용이 됩니다. 본질적으로 '시간이 정지된' 상태로, 모든 사용자가 프로젝트와 그 콘텐츠를 볼 수 있지만 수정이나 추가는 할 수 없습니다. 이를 통해 아카이브 당시 프로젝트 데이터의 무결성과 불변성을 보장합니다.
- **API 액세스 변경**: 프로젝트는 읽기 작업을 위해 Blue의 API를 통해 여전히 액세스할 수 있지만, 쓰기 작업(데이터 수정 또는 추가 시도)은 성공하지 않습니다. 이를 통해 API 상호작용을 통해서도 프로젝트의 읽기 전용 상태를 유지합니다.
