룩업 사용자 정의 필드를 사용하면 다른 프로젝트의 레코드에서 데이터를 가져와 단방향 가시성을 생성할 수 있습니다. [참조 사용자 정의 필드](/ko/docs/custom-fields/reference)가 프로젝트 간 연결을 설정하는 반면, 룩업은 이러한 연결을 사용하여 선택된 데이터를 표시합니다.

룩업 사용자 정의 필드는 항상 읽기 전용이며 특정 참조 사용자 정의 필드에 연결됩니다. 사용자가 참조 사용자 정의 필드를 사용하여 하나 이상의 레코드를 선택하면, 룩업 사용자 정의 필드는 해당 레코드의 데이터를 표시합니다.

룩업은 다음과 같은 데이터를 표시할 수 있습니다:

- **생성일**: 참조된 레코드가 생성된 시간.
- **업데이트일**: 참조된 레코드가 어떤 방식으로든 마지막으로 업데이트된 날짜와 시간.
- **마감일**: 참조 레코드의 시작/마감 날짜와 시간.
- **설명**: 참조된 레코드의 설명 필드.
- **목록**: 참조된 레코드가 속한 목록을 표시.
- **태그**: 참조된 레코드에 태그된 모든 태그.
- **담당자**: 참조된 레코드의 모든 담당자.
- **모든 사용자 정의 필드**: 참조된 레코드의 지원되는 모든 사용자 정의 필드.

![Duration](/docs/Lookup.png "Lookup")

룩업 필드에는 많은 옵션이 있습니다.

## 룩업의 룩업

여러 다른 프로젝트에서 데이터를 저장하고 참조하는 사용자 정의 필드를 생성할 수 있습니다. 이러한 사용자 정의 필드를 "룩업" 필드라고 합니다.

세 개의 프로젝트를 고려해 보겠습니다:

- **프로젝트 A**
- **프로젝트 B**
- **프로젝트 C**

프로젝트 B에 프로젝트 A와 프로젝트 C 모두에 관련된 중요한 정보나 리소스가 포함되어 있다고 가정합니다.

프로젝트 B의 정보를 프로젝트 A와 프로젝트 C에 중복 저장하는 대신 다음 단계를 따르세요:

- 프로젝트 A에서 프로젝트 B의 관련 레코드와 연결을 설정하기 위해 "참조" 사용자 정의 필드를 생성합니다.

  - 이 참조 사용자 정의 필드를 사용하면 참조하고 룩업하려는 프로젝트 B의 특정 레코드를 선택하고 연결할 수 있습니다.

- 프로젝트 A에서 참조 필드가 설정되면, 프로젝트 C에서 "룩업" 사용자 정의 필드를 생성합니다.

  - 이 룩업 사용자 정의 필드는 프로젝트 A의 참조 사용자 정의 필드에 연결됩니다.

  - 프로젝트 A의 참조 사용자 정의 필드를 사용하여 레코드를 선택하면, 프로젝트 C의 룩업 사용자 정의 필드는 프로젝트 B의 해당 참조된 레코드에서 관련 데이터를 자동으로 표시합니다.

## 룩업의 룩업 사용의 이점:

- 프로젝트 C를 프로젝트 B에 직접 연결할 필요가 없습니다.
- 프로젝트 C는 참조 및 룩업 필드를 통해 프로젝트 A와 프로젝트 B 간에 설정된 연결을 활용할 수 있습니다.
- 프로젝트 B의 관련 레코드에 대한 변경 사항이나 업데이트는 프로젝트 C에 표시되는 룩업 사용자 정의 필드 데이터에 자동으로 반영됩니다.
- 프로젝트 간에 정보를 수동으로 업데이트하거나 중복 저장할 필요가 없습니다.
