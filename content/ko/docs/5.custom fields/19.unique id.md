고유 ID 필드는 각 레코드에 대해 고유 식별자를 자동으로 생성합니다. Blue는 두 가지 유형의 고유 식별 시스템을 제공합니다:

## 1. 무작위 영숫자 ID (기본값)

표준 고유 ID 시스템은 각 레코드에 대해 고유한 영숫자 식별자를 생성합니다. 이는 각 레코드를 지속적으로 식별하는 영구적인 키를 생성하여 엄청난 가치를 제공합니다. 레코드가 삭제되더라도 해당 ID는 절대 중복되지 않습니다.

## 2. 순차 ID

사람이 읽기 쉬운 번호 형식의 식별자가 필요한 프로젝트를 위해 Blue는 순차 ID도 지원합니다. 이 기능을 활성화하면 시스템은 증분 번호(예: 1, 2, 3 또는 001, 002, 003과 같은 제로 패딩)를 생성하며, 접두사로 사용자 정의할 수 있습니다(예: TASK-001, TICKET-002).

## 사용 사례

두 ID 시스템 모두 강력한 기능을 제공합니다:

**일반적인 이점:**
- 고유 코드를 통해 외부 시스템을 레코드와 연결
- ID 참조를 추적하여 감사 추적 생성
- 레코드 이름이 변경되어도 ID를 통해 플랫폼에서 쉽게 레코드 검색

**순차 ID 특정 이점:**
- 팀 커뮤니케이션을 위한 사람이 읽기 쉬운 작업 참조
- 워크플로를 위한 일관된 번호 체계(예: 지원 티켓, 기능 요청)
- 숫자 식별자를 요구하는 외부 시스템과의 쉬운 통합
- 순차 접두사를 사용한 자동 제목 생성

![UniqueID 1](/docs/UniqueID_1.png "UniqueID 1")

두 ID 시스템 모두 기본적으로 각 레코드에 대한 불변의 "주민등록번호"를 설정합니다. 이러한 영구성과 특수성은 강력한 통합, 분석 및 식별 기능을 가능하게 합니다.

![UniqueID 2](/docs/UniqueID_2.png "UniqueID 2")

## 순차 ID 구성

고유 ID 사용자 정의 필드를 생성하거나 편집할 때 다음 옵션으로 순차 번호를 활성화할 수 있습니다:

- **순차 고유 ID 사용**: 무작위 영숫자 ID 대신 순차 번호를 활성화하는 토글
- **시퀀스 자릿수**: 제로 패딩을 위한 자릿수(예: 3자리 = 001, 002, 003)
- **시퀀스 시작 번호**: 계산을 시작할 번호(기본값: 1)
- **접두사**: 생성된 제목을 위한 선택적 텍스트 접두사(예: "TASK-"는 TASK-001, TASK-002를 생성)

**사용 가능한 총 고유 ID** <br/><br/>
Blue 고유 ID의 25개 문자 각각에 대해 36가지 가능성이 있습니다: <br/>
- 26개 대문자(A-Z) <br/>
- 10개 숫자(0-9) <br/>

이는 총 8.08×10^35가지 가능성, 즉 더 정확히 말하면: 808,281,277,464,764,060,643,139,600,456,536,293,376가지를 의미합니다 <br/><br/>
이를 비교해보면, 지구상의 모든 모래알(7.5*10^18)에 대해 수십 경(1.08×10^17)개의 고유 ID가 있습니다!
::

## 무작위 영숫자 ID 구성

프로젝트 관리자는 모든 무작위 영숫자 고유 ID에 대해 사용자 정의 접두사를 구성할 수 있습니다.

프로젝트 관리자는 모든 고유 ID에 대해 선택적 접두사를 설정할 수 있습니다

**순차 ID 기술적 세부사항** <br/><br/>
순차 ID는 프로젝트 범위로, 각 프로젝트가 자체 시퀀스 카운터를 유지합니다. 레코드가 프로젝트 간에 복사되거나 이동될 때, 시스템은 일치하는 사용자 정의 필드 구성을 기반으로 시퀀스 할당을 지능적으로 처리합니다. 시퀀스 생성은 동시 환경에서 중복 번호를 방지하기 위해 원자적이고 스레드 안전합니다.
