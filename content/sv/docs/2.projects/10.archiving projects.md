---
title: Arkivera projekt
description: Frys projekt när de är färdiga för regelefterlevnad.
---

I många organisationer är det avgörande att behålla data från slutförda projekt för revisionssyften eller för att återbesöka projektet i framtiden. Att ackumulera hundratals slutförda projekt kan dock göra Blue-gränssnittet rörigt, vilket utmanar navigering och hantering av aktuella projekt.

Detta är varför Blue erbjuder möjligheten för [Projektadministratörer](/docs/user-management/roles/project-administrator) att arkivera projekt när de är färdiga eller inte längre behövs.

![Archiving Project 1](/docs/ArchivingProject_1.png "Archiving Project 1")

Arkivera enkelt vilket projekt som helst

När ett projekt arkiveras händer flera saker.

- **Omplacering till 'Arkiverade projekt'**: Projektet flyttas från huvudområdet 'Projekt' i sidofältet till en särskild sektion 'Arkiverade projekt'. Detta hjälper till att upprätthålla ett rent och organiserat huvudgränssnitt.
- **Synlig arkiveringsstatus**: I den övre balken visas en 'Arkiverad' etikett bredvid projektnamnet, vilket tydligt indikerar dess status.
- **Skrivskyddad åtkomst**: Arkiverade projekt blir skrivskyddade. De är i huvudsak 'frysta i tiden', vilket betyder att alla användare kan se projektet och dess innehåll, men inga ändringar eller tillägg kan göras. Detta säkerställer integriteten och oföränderligheten av projektdata som de var vid tidpunkten för arkivering.
- **API-åtkomständringar**: Även om projektet fortfarande kan nås genom Blues API för läsoperationer, kommer alla skrivoperationer (försök att ändra eller lägga till data) inte att lyckas. Detta upprätthåller projektets skrivskyddade status även genom API-interaktioner.
