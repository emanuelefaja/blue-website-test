Fältet Unikt ID genererar automatiskt en unik identifierare för varje post. Blue erbjuder två typer av unika identifieringssystem:

## 1. Slumpmässiga alfanumeriska ID:n (Standard)

Det vanliga systemet för Unikt ID genererar en unik alfanumerisk identifierare för varje post. Detta ger enormt värde genom att skapa beständiga nycklar som konsekvent identifierar varje post. Även om en post tas bort kommer dess ID aldrig att dupliceras.

## 2. Sekventiella ID:n

För projekt som kräver läsbara, numrerade identifierare stöder Blue även sekventiella ID:n. När detta system är aktiverat genererar det inkrementella nummer (t.ex. 1, 2, 3, eller med nollpadding som 001, 002, 003) som kan anpassas med prefix (t.ex. TASK-001, TICKET-002).

## Användningsområden

Båda ID-systemen möjliggör kraftfulla funktioner:

**Allmänna fördelar:**
- Associera externa system till poster via unika koder
- Skapa revisionsspår genom att spåra ID-referenser
- Enkelt söka efter poster i plattformen via ID:t, även om postens namn ändras

**Specifika fördelar med sekventiella ID:n:**
- Läsbara uppgiftsreferenser för teamkommunikation
- Konsekventa numreringssystem för arbetsflöden (t.ex. supportärenden, funktionsförfrågningar)
- Enkel integration med externa system som förväntar sig numeriska identifierare
- Automatisk titelgenerering med sekventiella prefix

Båda ID-systemen etablerar i princip oföränderliga "personnummer" för varje post. Denna beständighet och specificitet möjliggör kraftfulla integrationer, analyser och identifieringsmöjligheter.

![UniqueID 2](/docs/UniqueID_2.png "UniqueID 2")

## Konfiguration av sekventiella ID:n

När du skapar eller redigerar ett anpassat fält för Unikt ID kan du aktivera sekventiell numrering med dessa alternativ:

- **Använd sekventiellt unikt ID**: Växla för att aktivera sekventiell numrering istället för slumpmässiga alfanumeriska ID:n
- **Sekvensiffror**: Antal siffror för nollpadding (t.ex. 3 siffror = 001, 002, 003)
- **Startpunkt för sekvens**: Numret att börja räkna från (standard: 1)
- **Prefix**: Valfritt textprefix för genererade titlar (t.ex. "TASK-" skapar TASK-001, TASK-002)

**Totalt antal tillgängliga unika ID:n** <br/><br/>
Det finns 36 möjligheter för var och en av de 25 tecknen i ett Blue Unikt ID: <br/>
- 26 versaler (A-Z) <br/>
- 10 siffror (0-9) <br/>

**Tekniska detaljer för sekventiella ID:n** <br/><br/>
Sekventiella ID:n är projektomfattande, vilket innebär att varje projekt upprätthåller sin egen sekvensräknare. När poster kopieras eller flyttas mellan projekt hanterar systemet intelligent sekvensstilldelning baserat på matchande anpassade fältkonfigurationer. Sekvensgenerering är atomisk och trådsäker för att förhindra duplicerade nummer i samtidiga miljöer.
