នៅក្នុងអង្គការជាច្រើន ការរក្សានូវទិន្នន័យពីគម្រោងដែលបានបញ្ចប់គឺមានសារៈសំខាន់សម្រាប់គោលបំណងសវនកម្ម ឬសម្រាប់ការពិនិត្យគម្រោងម្ដងទៀតនាពេលអនាគត។ ទោះជាយ៉ាងណាក៏ដោយ ការប្រមូលផ្ដុំគម្រោងដែលបានបញ្ចប់រាប់រយអាចធ្វើឱ្យចំណុចប្រទាក់ Blue ត្រូវបានកកកុញ ដែលបន្លំការរុករក និងការគ្រប់គ្រងគម្រោងបច្ចុប្បន្ន។

នេះជាមូលហេតុដែល Blue ផ្ដល់លទ្ធភាពឱ្យ[អ្នកគ្រប់គ្រងគម្រោង](/docs/user-management/roles/project-administrator)រក្សាទុកគម្រោងបន្ទាប់ពីពួកវាបានបញ្ចប់ ឬមិនត្រូវការទៀត។

![Archiving Project 1](/docs/ArchivingProject_1.png "Archiving Project 1")

រក្សាទុកគម្រោងណាមួយយ៉ាងងាយស្រួល

នៅពេលគម្រោងមួយត្រូវបានរក្សាទុក រឿងជាច្រើនកើតឡើង។

- **ការផ្លាស់ទីទៅ 'គម្រោងដែលបានរក្សាទុក'**: គម្រោងត្រូវបានផ្លាស់ទីពីតំបន់មេ 'គម្រោង' នៅលើរបារចំហៀងទៅផ្នែកកំណត់ 'គម្រោងដែលបានរក្សាទុក'។ នេះជួយធានានូវចំណុចប្រទាក់មេស្អាត និងមានរៀបរយ។
- **ស្ថានភាពរក្សាទុកដែលអាចមើលឃើញ**: នៅលើរបារកំពូល ស្លាក 'បានរក្សាទុក' លេចឡើងនៅជាប់ឈ្មោះគម្រោង បង្ហាញឱ្យឃើញច្បាស់នូវស្ថានភាពរបស់វា។
- **ការចូលប្រើតែអាន**: គម្រោងដែលបានរក្សាទុកក្លាយជាតែអានប៉ុណ្ណោះ។ ពួកវាគឺ 'បានបង្កកនៅក្នុងពេលវេលា' មានន័យថាអ្នកប្រើប្រាស់ទាំងអស់អាចមើលគម្រោង និងមាតិការបស់វាបាន ប៉ុន្តែមិនអាចធ្វើការកែប្រែ ឬបន្ថែមបានទេ។ នេះធានានូវភាពត្រឹមត្រូវ និងភាពមិនអាចប្ដូរបាននៃទិន្នន័យគម្រោងដូចដែលពួកវាមាននៅពេលរក្សាទុក។
- **ការផ្លាស់ប្ដូរការចូលប្រើ API**: ទោះបីជាគម្រោងនៅតែអាចចូលប្រើបានតាមរយៈ API របស់ Blue សម្រាប់ប្រតិបត្តិការអានក៏ដោយ ប្រតិបត្តិការសរសេរណាមួយ (ការព្យាយាមកែប្រែ ឬបន្ថែមទិន្នន័យ) នឹងមិនជោគជ័យទេ។ នេះរក្សាស្ថានភាពតែអានរបស់គម្រោងសូម្បីតែតាមរយៈអន្តរកម្ម API។
