វាលកំណត់ផ្ទាល់ខ្លួន Lookup អនុញ្ញាតឱ្យអ្នកនាំចូលទិន្នន័យពីកំណត់ត្រាក្នុងគម្រោងផ្សេងទៀត ដោយបង្កើតភាពមើលឃើញតែមួយទិសដៅ។ ខណៈពេលដែល [វាលកំណត់ផ្ទាល់ខ្លួន Reference](/km/docs/custom-fields/reference) បង្កើតការតភ្ជាប់អ្នកគម្រោង Lookups ប្រើការតភ្ជាប់ទាំងនោះដើម្បីបង្ហាញទិន្នន័យដែលបានជ្រើសរើស។

វាលកំណត់ផ្ទាល់ខ្លួន Lookup តែងតែអាចអានបានតែប៉ុណ្ណោះ ហើយត្រូវបានតភ្ជាប់ទៅនឹងវាលកំណត់ផ្ទាល់ខ្លួន Reference ជាក់លាក់មួយ។ នៅពេលដែលអ្នកប្រើជ្រើសរើសកំណត់ត្រាមួយ ឬច្រើនដោយប្រើវាលកំណត់ផ្ទាល់ខ្លួន Reference វាលកំណត់ផ្ទាល់ខ្លួន Lookup នឹងបង្ហាញទិន្នន័យពីកំណត់ត្រាទាំងនោះ។

Lookups អាចបង្ហាញទិន្នន័យដូចជា៖

- **បង្កើតនៅ**: ពេលណាដែលកំណត់ត្រាដែលបានយោងត្រូវបានបង្កើត។
- **កែប្រែនៅ**: កាលបរិច្ឆេទ និងពេលវេលាចុងក្រោយដែលកំណត់ត្រាដែលបានយោងត្រូវបានកែប្រែតាមវិធីណាមួយ។
- **កាលបរិច្ឆេទកំណត់**: កាលបរិច្ឆេទចាប់ផ្តើម/កំណត់ និងពេលវេលារបស់កំណត់ត្រាយោង។
- **ការពិពណ៌នា**: វាលការពិពណ៌នារបស់កំណត់ត្រាដែលបានយោង។
- **បញ្ជី**: បង្ហាញបញ្ជីដែលកំណត់ត្រាដែលបានយោងជាប់ពាក់ព័ន្ធ។
- **ស្លាក**: ស្លាកណាមួយដែលកំណត់ត្រាដែលបានយោងត្រូវបានដាក់ស្លាក។
- **អ្នកទទួលបានការងារ**: អ្នកទទួលបានការងារណាមួយសម្រាប់កំណត់ត្រាដែលបានយោង។
- **វាលកំណត់ផ្ទាល់ខ្លួនណាមួយ**: វាលកំណត់ផ្ទាល់ខ្លួនដែលត្រូវបានគាំទ្រណាមួយពីកំណត់ត្រាដែលបានយោង។

## Lookup a Lookup

អ្នកអាចបង្កើតវាលកំណត់ផ្ទាល់ខ្លួនដើម្បីរក្សាទុក និងយោងទិន្នន័យនៅទូទាំងគម្រោងផ្សេងៗជាច្រើន។ វាលកំណត់ផ្ទាល់ខ្លួនទាំងនេះត្រូវបានហៅថា "Lookup" fields។

តោះយកករណីគម្រោងបីរូប៖

- **គម្រោង A**
- **គម្រោង B**
- **គម្រោង C**

សន្មតថាគម្រោង B មានព័ត៌មានសំខាន់ៗ ឬធនធានដែលពាក់ព័ន្ធទាំងគម្រោង A និងគម្រោង C។

ជំនួសឱ្យការបន្ទាត់ព័ត៌មានពីគម្រោង B ទៅគម្រោង A និងគម្រោង C សូមធ្វើតាមជំហានទាំងនេះ៖

- នៅក្នុងគម្រោង A បង្កើតវាលកំណត់ផ្ទាល់ខ្លួន "Reference" ដើម្បីបង្កើតការតភ្ជាប់ជាមួយកំណត់ត្រាពាក់ព័ន្ធពីគម្រោង B។

  - វាលកំណត់ផ្ទាល់ខ្លួន Reference នេះអនុញ្ញាតឱ្យអ្នកជ្រើសរើស និងតភ្ជាប់ទៅកំណត់ត្រាជាក់លាក់នៅក្នុងគម្រោង B ដែលអ្នកចង់យោង និង lookup។

- នៅពេលដែលវាល Reference ត្រូវបានរៀបចំនៅក្នុងគម្រោង A បង្កើតវាលកំណត់ផ្ទាល់ខ្លួន "Lookup" នៅក្នុងគម្រោង C។

  - វាលកំណត់ផ្ទាល់ខ្លួន Lookup នេះនឹងត្រូវបានតភ្ជាប់ទៅវាលកំណត់ផ្ទាល់ខ្លួន Reference នៅក្នុងគម្រោង A។

  - នៅពេលដែលអ្នកជ្រើសរើសកំណត់ត្រាដោយប្រើវាលកំណត់ផ្ទាល់ខ្លួន Reference នៅក្នុងគម្រោង A វាលកំណត់ផ្ទាល់ខ្លួន Lookup នៅក្នុងគម្រោង C នឹងបង្ហាញទិន្នន័យពាក់ព័ន្ធពីកំណត់ត្រាដែលបានយោងទាំងនោះនៅក្នុងគម្រោង B ដោយស្វ័យប្រវត្តិ។

## អត្ថប្រយោជន៍នៃការប្រើ Lookup a Lookup៖

- អ្នកមិនចាំបាច់តភ្ជាប់គម្រោង C ទៅគម្រោង B ដោយផ្ទាល់ទេ។
- គម្រោង C អាចប្រើប្រាស់ការតភ្ជាប់ដែលបានបង្កើតរវាងគម្រោង A និងគម្រោង B តាមរយៈវាល Reference និង Lookup។
- ការផ្លាស់ប្តូរ ឬការកែប្រែណាមួយដែលធ្វើឡើងចំពោះកំណត់ត្រាពាក់ព័ន្ធនៅក្នុងគម្រោង B នឹងឆ្លុះបញ្ចាំងដោយស្វ័យប្រវត្តិនៅក្នុងទិន្នន័យវាលកំណត់ផ្ទាល់ខ្លួន Lookup ដែលបង្ហាញនៅក្នុងគម្រោង C។
- មិនចាំបាច់មានការកែប្រែដោយដៃ ឬការបន្ទាត់ព័ត៌មាននៅទូទាំងគម្រោងទេ។
