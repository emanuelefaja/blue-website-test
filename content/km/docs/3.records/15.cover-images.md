## ទិដ្ឋភាពទូទៅ

រូបភាពគម្របអនុញ្ញាតឱ្យអ្នកបន្ថែមបរិបទដែលមានភាពជាក់ស្តែងទៅកំណត់ត្រារបស់អ្នកដោយការបង្ហាញការមើលមុនរូបភាពដោយផ្ទាល់នៅលើកាតកំណត់ត្រា។ លក្ខណៈពិសេសនេះស្រង់យករូបភាពដោយស្វ័យប្រវត្តិពីប្រភពផ្សេងៗនៅក្នុងកំណត់ត្រារបស់អ្នកហើយបង្ហាញពួកវាជារូបភាពតូច ដោយធ្វើឱ្យវាកាន់តែងាយស្រួលក្នុងការកំណត់និងរៀបចំមាតិកាដែលមានភាពជាក់ស្តែងក្នុងមួយភ្លែត។

មិនថាអ្នកកំពុងគ្រប់គ្រងគម្រោងរចនា ប្រតិទិនមាតិកា កាតាឡុកផលិតផល ឬលំហូរការងារណាមួយដែលពាក់ព័ន្ធនឹងទ្រព្យសម្បត្តិដែលមានភាពជាក់ស្តែងទេ រូបភាពគម្របជួយអ្នកស្កេនកំណត់ត្រាយ៉ាងរហ័សនិងស្វែងរកអ្វីដែលអ្នកកំពុងស្វែងរក។ លក្ខណៈពិសេសនេះរួមបញ្ចូលការតាមដានវឌ្ឍនភាពក្នុងពេលជាក់ស្តែងនិងជម្រើសបង្ហាញដែលមានភាពបត់បែនដើម្បីផ្គូផ្គងនឹងតម្រូវការលំហូរការងាររបស់អ្នក។

## លក្ខណៈពិសេសចម្បង

- **ការស្រង់យករូបភាពដោយស្វ័យប្រវត្តិ**: Blue ស្វែងរកនិងបង្ហាញរូបភាពដោយស្វ័យប្រវត្តិពីប្រភពដែលអ្នកបានជ្រើសរើស
- **ប្រភពរូបភាពច្រើន**: ទាញយករូបភាពពីការពិពណ៌នាកំណត់ត្រា មតិយោបល់ ឬវាលឯកសារផ្ទាល់ខ្លួន
- **ជម្រើសការបង្ហាញ**: ជ្រើសរើសរបៀបដែលរូបភាពបង្ហាញជាមួយនឹង Cover ឬ Contain
- **ការជ្រើសរើសដ៏ឆ្លាតវៃ**: កំណត់រចនាសម្ព័ន្ធថាតើត្រូវប្រើរូបភាពដំបូងឬចុងក្រោយពីប្រភពរបស់អ្នក
- **ការអាប់ដេតក្នុងពេលជាក់ស្តែង**: រូបភាពគម្របអាប់ដេតដោយស្វ័យប្រវត្តិនៅពេលមាតិកាកំណត់ត្រាផ្លាស់ប្តូរ
- **ការតាមដានវឌ្ឍនភាពផ្ទាល់**: តាមដានដំណើរការក្នុងបរិមាណច្រើនជាមួយនឹងការជាវវឌ្ឍនភាពក្នុងពេលជាក់ស្តែង

![ទិដ្ឋភាពទូទៅរូបភាពគម្រប](/docs/records/cover-images/overview.png "ទិដ្ឋភាពទូទៅរូបភាពគម្រប")

រូបភាពគម្របធ្វើឱ្យកំណត់ត្រារបស់អ្នកអាចកំណត់បានភ្លាមៗ

## របៀបដែលវាដំណើរការ

### ការបើករូបភាពគម្រប

រូបភាពគម្របត្រូវបានកំណត់រចនាសម្ព័ន្ធនៅកម្រិតគម្រោងដោយអ្នកគ្រប់គ្រងគម្រោង៖

1. បើកគម្រោងរបស់អ្នកហើយនាវិគេសទៅកាន់ **ការកំណត់គម្រោង**
2. ជ្រើសរើស **ការកំណត់កំណត់ត្រា**
3. ស្វែងរកផ្នែក **ជម្រើសគម្របរូបភាព**
4. ប្តូរ **បើករូបភាពគម្រប** ដើម្បីសកម្មលក្ខណៈពិសេសនេះ

### ជម្រើសកំណត់រចនាសម្ព័ន្ធ

#### ប្រភពរូបភាព
ជ្រើសរើសកន្លែងដែល Blue គួរស្វែងរករូបភាព៖
- **ការពិពណ៌នា**: ស្រង់យករូបភាពដែលបានបង្កប់នៅក្នុងការពិពណ៌នាកំណត់ត្រា (លំនាំដើម)
- **មតិយោបល់**: ទាញយករូបភាពដំបូង/ចុងក្រោយពីមតិយោបល់កំណត់ត្រា
- **វាលផ្ទាល់ខ្លួន**: ប្រើឯកសារពីវាលផ្ទាល់ខ្លួនប្រភេទឯកសារជាក់លាក់ (គាំទ្រប្រភេទឯកសារណាមួយ)

#### ការជ្រើសរើសរូបភាព
នៅពេលមានរូបភាពជាច្រើន ជ្រើសរើសរូបភាពណាដែលត្រូវបង្ហាញ៖
- **រូបភាពដំបូង**: ប្រើរូបភាពដំបូងដែលបានរកឃើញ (លំនាំដើម)
- **រូបភាពចុងក្រោយ**: ប្រើរូបភាពថ្មីបំផុត

#### ការបង្ហាញ
ត្រួតពិនិត្យរបៀបដែលរូបភាពបង្ហាញនៅក្នុងតំបន់មើលមុន៖
- **Cover**: រូបភាពបំពេញតំបន់មើលមុនទាំងមូល អាចកាត់គែមដើម្បីរក្សាសមាមាត្រ (លំនាំដើម)
- **Contain**: បង្ហាញរូបភាពពេញនៅក្នុងតំបន់មើលមុន អាចមានកន្លែងទំនេរ

![ការកំណត់រូបភាពគម្រប](/docs/records/cover-images/settings.png "ការកំណត់រូបភាពគម្រប")

កំណត់រចនាសម្ព័ន្ធរូបភាពគម្របដើម្បីផ្គូផ្គងនឹងតម្រូវការលំហូរការងាររបស់អ្នក

## ករណីប្រើប្រាស់

**គម្រោងរចនា** - បង្ហាញការរៀបចំនិងការធ្វើឱ្យប្រសើរឡើងដោយផ្ទាល់នៅលើកាតកិច្ចការ ដោយធ្វើឱ្យវាងាយស្រួលមើលស្ថានភាពបច្ចុប្បន្នរបស់កិច្ចការរចនានីមួយៗដោយមិនចាំបាច់បើកកំណត់ត្រាបុគ្គល។

**ការរៀបចំមាតិកា** - សម្រាប់ប្រតិទិនមាតិកានិងលំហូរការងារវិចារណកថា បង្ហាញរូបភាពសំខាន់ៗឬរូបភាពតូចដើម្បីមើលបណ្តាញលំហូរមាតិការបស់អ្នកក្នុងមួយភ្លែត។

**ការគ្រប់គ្រងផលិតផល** - បង្ហាញរូបថតផលិតផលនៅលើកំណត់ត្រាសារពើភណ្ឌឬសំណើលក្ខណៈពិសេស ជួយក្រុមកំណត់គ្រឿងរហ័សក្នុងអំឡុងពេលពិភាក្សា។

**អចលនទ្រព្យ** - បង្ហាញរូបថតអចលនទ្រព្យនៅលើកំណត់ត្រាបញ្ជី អនុញ្ញាតឱ្យភ្នាក់ងារមើលដោយលំហូរអចលនទ្រព្យតាមរបៀបដែលមានភាពជាក់ស្តែង។

**ការរៀបចំព្រឹត្តិការណ៍** - បង្ហាញរូបថតកន្លែង គំនិតរចនា ឬប្លង់ព្រឹត្តិការណ៍នៅលើកិច្ចការរៀបចំសម្រាប់ការរៀបចំដែលមានភាពជាក់ស្តែងប្រសើរជាង។

## ការដំឡើងរូបភាពគម្រប

### ជំហានទី១៖ ជ្រើសរើសប្រភពរូបភាពរបស់អ្នក

ពិចារណាកន្លែងដែលរូបភាពមានដោយធម្មជាតិនៅក្នុងលំហូរការងាររបស់អ្នក៖
- ប្រសិនបើរូបភាពជាផ្នែកនៃការពិពណ៌នាកិច្ចការឬសង្ខេប ប្រើ **ការពិពណ៌នា**
- ប្រសិនបើរូបភាពត្រូវបានបន្ថែមតាមរយៈការសហការក្រុម ប្រើ **មតិយោបល់**
- ប្រសិនបើអ្នកមានវាលជាក់លាក់សម្រាប់រូបភាព ប្រើ **វាលផ្ទាល់ខ្លួន**

### ជំហានទី២៖ កំណត់រចនាសម្ព័ន្ធការកំណត់បង្ហាញ

1. ជ្រើសរើស **ការបង្ហាញរូបភាព** ដែលអ្នកចូលចិត្តផ្អែកលើប្រភេទរូបភាពរបស់អ្នក៖
   - ប្រើ **Cover** សម្រាប់រូបថតនិងរចនាដែលការកាត់អាចទទួលយកបាន
   - ប្រើ **Contain** សម្រាប់ដ្យាក្រាមឬរូបថតអេក្រង់ដែលអ្នកត្រូវការមើលអ្វីគ្រប់យ៉ាង

2. ជ្រើសរើស **ការជ្រើសរើសរូបភាព** ផ្អែកលើលំហូរការងាររបស់អ្នក៖
   - ប្រើ **ដំបូង** សម្រាប់លំហូរការងារដែលរូបភាពចម្បងត្រូវបានបន្ថែមដំបូង
   - ប្រើ **ចុងក្រោយ** សម្រាប់លំហូរការងារដែលរូបភាពថ្មីបំផុតមានភាពសម្រាប់បំផុត

### ជំហានទី៣៖ ដំណើរការកំណត់ត្រាដែលមានស្រាប់

នៅពេលអ្នកបើករូបភាពគម្របដំបូងឬផ្លាស់ប្តូរកំណត់រចនាសម្ព័ន្ធ Blue ដំណើរការកំណត់ត្រារបស់អ្នកដោយស្វ័យប្រវត្តិ៖
1. ការអាប់ដេតវឌ្ឍនភាពក្នុងពេលជាក់ស្តែងបង្ហាញស្ថានភាពដំណើរការ
2. កំណត់ត្រាត្រូវបានដំណើរការក្នុងបណ្តុំ (៥០ កំណត់ត្រាក្នុងមួយបណ្តុំ) សម្រាប់ការអនុវត្តល្អបំផុត
3. អ្នកអាចបន្តធ្វើការខណៈពេលដំណើរការកំពុងកើតឡើងនៅផ្ទៃខាងក្រោយ
4. បញ្ជីនឹងធ្វើការធ្វើឱ្យស្រស់ដោយស្វ័យប្រវត្តិនៅពេលដំណើរការបាន​បំពេញ
5. ដំណើរការបង្កដោយស្វ័យប្រវត្តិនៅពេលការផ្លាស់ប្តូរកំណត់រចនាសម្ព័ន្ធត្រូវបានរក្សាទុក

## ព័ត៌មានលម្អិតបច្ចេកទេស

### ទ្រង់ទ្រាយដែលគាំទ្រ
- នៅពេលប្រើប្រភពការពិពណ៌នាឬមតិយោបល់៖
  - រូបភាព JPEG/JPG
  - រូបភាព PNG (រួមទាំងតម្លាភាព)
  - រូបភាព GIF (បង្ហាញស្ថិតថេរ)
- នៅពេលប្រើប្រភពវាលផ្ទាល់ខ្លួន៖
  - ប្រភេទឯកសារណាមួយអាចត្រូវបានបង្ហាញ (រូបភាព PDFs ឯកសារ)
  - ឯកសារមិនមែនរូបភាពបង្ហាញរូបតំណាងប្រភេទឯកសារសមរម្យ
- ឯកសារទាំងអស់ត្រូវតែបានអាប់ឡូដទៅ Blue (URL ខាងក្រៅមិនត្រូវបានគាំទ្រ)

### កម្រិត
- អតិបរមា ៦០០០ កំណត់ត្រាក្នុងមួយគម្រោងសម្រាប់ការបង្កើតគម្របក្នុងបរិមាណច្រើន
- មានតែវាលផ្ទាល់ខ្លួនប្រភេទឯកសារប៉ុណ្ណោះដែលអាចត្រូវបានប្រើជាប្រភព
- រូបភាពគម្របមានតែការមើលប៉ុណ្ណោះហើយមិនអាចកែសម្រួលដោយផ្ទាល់

### ការអាប់ដេតដោយស្វ័យប្រវត្តិ
គម្របបង្កើតឡើងវិញដោយស្វ័យប្រវត្តិនៅពេល៖
- ការពិពណ៌នាកំណត់ត្រាត្រូវបានកែសម្រួល (ប្រសិនបើប្រើប្រភពការពិពណ៌នា)
- មតិយោបល់ថ្មីជាមួយនឹងរូបភាពត្រូវបានបន្ថែម (ប្រសិនបើប្រើប្រភពមតិយោបល់)
- ឯកសារនៅក្នុងវាលផ្ទាល់ខ្លួនត្រូវបានអាប់ដេត (ប្រសិនបើប្រើប្រភពវាលផ្ទាល់ខ្លួន)
- ការកំណត់រចនាសម្ព័ន្ធគម្របត្រូវបានផ្លាស់ប្តូរ
- ជម្រើសការបង្ហាញត្រូវបានកែប្រែ

### ការចូលប្រើ API
រូបភាពគម្របត្រូវបានរួមបញ្ចូលពេញលេញជាមួយ GraphQL API៖
- កំណត់រចនាសម្ព័ន្ធតាមរយៈ `setTodoCoverConfig` mutation
- ចូលដំណើរការទិន្នន័យគម្របតាមរយៈវាល `cover` នៅលើប្រភេទ Todo
- តាមដានវឌ្ឍនភាពជាមួយ `subscribeToCoverGenerationProgress` subscription
- URL រូបភាពតូចបង្កើតដោយស្វ័យប្រវត្តិនៅ `/uploads/{uid}/thumbnail/cover`


## ការរួមបញ្ចូលជាមួយនឹងលក្ខណៈពិសេសផ្សេងទៀត

រូបភាពគម្របដំណើរការយ៉ាងរលូនជាមួយនឹងលក្ខណៈពិសេស Blue ផ្សេងទៀត៖
- **[ទម្រង់](/km/docs/projects/forms)**: រូបភាពដែលបានដាក់ស្នើអាចក្លាយជាគម្របដោយស្វ័យប្រវត្តិ
- **[API](/km/docs/integrations/api)**: ការគាំទ្រ GraphQL API ពេញលេញសម្រាប់កំណត់រចនាសម្ព័ន្ធនិងការចូលដំណើរការទិន្នន័យ
- **[វាលផ្ទាល់ខ្លួន](/km/docs/projects/custom-fields)**: វាលប្រភេទឯកសារណាមួយអាចបម្រើជាប្រភពគម្រប
