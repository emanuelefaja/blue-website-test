---
title: 封存專案
description: 完成專案後凍結以符合規範。
---

在許多組織中，保留已完成專案的資料對於審計目的或未來重新檢視專案至關重要。然而，累積數百個已完成的專案可能會使 Blue 介面變得雜亂，對導航和當前專案管理造成挑戰。

這就是為什麼 Blue 提供[專案管理員](/docs/user-management/roles/project-administrator)在專案完成或不再需要時封存專案的功能。

![Archiving Project 1](/docs/ArchivingProject_1.png "Archiving Project 1")

輕鬆封存任何專案

當專案被封存時，會發生多件事。

- **重新配置到「已封存專案」**：專案從側邊欄的主要「專案」區域移動到指定的「已封存專案」部分。這有助於維持乾淨且有組織的主介面。
- **可見的封存狀態**：在頂部欄中，專案名稱旁會出現「已封存」標籤，清楚地標示其狀態。
- **唯讀存取**：封存的專案變為唯讀。它們本質上是「時間凍結」，意味著所有使用者都可以檢視專案及其內容，但無法進行任何修改或新增。這確保了專案資料在封存時的完整性和不可變性。
- **API 存取變更**：雖然專案仍可透過 Blue 的 API 進行讀取操作存取，但任何寫入操作（嘗試修改或新增資料）都不會成功。這即使透過 API 互動也能維持專案的唯讀狀態。
