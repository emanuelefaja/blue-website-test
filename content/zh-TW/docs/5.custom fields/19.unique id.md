唯一 ID 欄位會自動為每筆記錄生成唯一識別碼。Blue 提供兩種唯一識別系統：

## 1. 隨機字母數字 ID（預設）

標準唯一 ID 系統為每筆記錄生成唯一的字母數字識別碼。這帶來巨大價值，創建持久性鍵值來持續識別每筆記錄。即使記錄被刪除，其 ID 也永遠不會重複。

## 2. 序列 ID

對於需要人類可讀、編號識別碼的專案，Blue 也支援序列 ID。啟用後，此系統會生成遞增數字（例如 1、2、3，或使用零填充如 001、002、003），並可使用前綴進行自訂（例如 TASK-001、TICKET-002）。

## 使用案例

兩種 ID 系統都能實現強大功能：

**一般優勢：**
- 透過唯一代碼將外部系統與記錄關聯
- 透過追蹤 ID 參照建立稽核軌跡
- 在平台中透過 ID 輕鬆搜尋記錄，即使記錄名稱變更也不受影響

**序列 ID 特定優勢：**
- 便於團隊溝通的人類可讀任務參照
- 工作流程的一致編號方案（例如支援票券、功能請求）
- 與期望數字識別碼的外部系統輕鬆整合
- 使用序列前綴自動生成標題

![UniqueID 1](/docs/UniqueID_1.png "UniqueID 1")

兩種 ID 系統本質上都為每筆記錄建立不可變的「身份證號碼」。這種永久性和特異性使得強大的整合、分析和識別功能成為可能。

![UniqueID 2](/docs/UniqueID_2.png "UniqueID 2")

## 序列 ID 配置

在建立或編輯唯一 ID 自訂欄位時，您可以使用以下選項啟用序列編號：

- **使用序列唯一 ID**：切換啟用序列編號而非隨機字母數字 ID
- **序列位數**：零填充的位數（例如 3 位數 = 001、002、003）
- **序列起始號碼**：開始計數的號碼（預設：1）
- **前綴**：生成標題的可選文字前綴（例如「TASK-」建立 TASK-001、TASK-002）

**可用的唯一 ID 總數** <br/><br/>
Blue 唯一 ID 的 25 個字元中，每個都有 36 種可能性： <br/>
- 26 個大寫字母（A-Z） <br/>
- 10 個數字（0-9） <br/>

這意味著總共有 8.08×10^35 種可能性，或者更準確地說：808,281,277,464,764,060,643,139,600,456,536,293,376 <br/><br/>
相比之下，地球上每一粒沙子（7.5*10^18），都有數十兆（1.08×10^17）個唯一 ID！
::

**序列 ID 技術詳情** <br/><br/>
序列 ID 是專案範圍的，意味著每個專案維護自己的序列計數器。當記錄在專案間複製或移動時，系統會根據匹配的自訂欄位配置智慧地處理序列分配。序列生成是原子性和線程安全的，以防止在並發環境中產生重複號碼。
