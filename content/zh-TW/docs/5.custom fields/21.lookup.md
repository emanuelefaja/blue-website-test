查找自定義欄位允許您從其他專案的記錄中匯入資料，建立單向可見性。雖然[參考自定義欄位](/zh-TW/docs/custom-fields/reference)建立跨專案連接，查找欄位使用這些連接來顯示選定的資料。

查找自定義欄位始終是唯讀的，並連接到特定的參考自定義欄位。當使用者使用參考自定義欄位選擇一個或多個記錄時，查找自定義欄位將顯示來自這些記錄的資料。

查找欄位可以顯示的資料包括：

- **建立時間**：參考記錄建立的時間。
- **更新時間**：參考記錄最後以任何方式更新的日期和時間。
- **到期日**：參考記錄的開始/到期日期和時間。
- **描述**：參考記錄的描述欄位。
- **清單**：顯示參考記錄所屬的清單。
- **標籤**：標記在參考記錄上的任何標籤。
- **受指派人**：指派給參考記錄的任何受指派人。
- **任何自定義欄位**：來自參考記錄的任何支援的自定義欄位。

## 查找中的查找

您可以建立自定義欄位來儲存和參考跨多個不同專案的資料。這些自定義欄位稱為「查找」欄位。

讓我們考慮三個專案：

- **專案 A**
- **專案 B**
- **專案 C**

假設專案 B 包含一些對專案 A 和專案 C 都相關的重要資訊或資源。

與其將專案 B 的資訊複製到專案 A 和專案 C 中，請按照以下步驟操作：

- 在專案 A 中，建立一個「參考」自定義欄位以建立與專案 B 中相關記錄的連接。

  - 此參考自定義欄位允許您選擇並連結到專案 B 中您想要參考和查找的特定記錄。

- 在專案 A 中設定參考欄位後，在專案 C 中建立一個「查找」自定義欄位。

  - 此查找自定義欄位將連接到專案 A 中的參考自定義欄位。

  - 當您在專案 A 中使用參考自定義欄位選擇記錄時，專案 C 中的查找自定義欄位將自動顯示來自專案 B 中這些參考記錄的相關資料。

## 使用查找中查找的好處：

- 您不需要直接將專案 C 連結到專案 B。
- 專案 C 可以通過參考和查找欄位利用專案 A 和專案 B 之間建立的連接。
- 對專案 B 中相關記錄所做的任何更改或更新都會自動反映在專案 C 中顯示的查找自定義欄位資料中。
- 無需手動更新或跨專案複製資訊。
