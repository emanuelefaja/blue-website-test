I campi personalizzati Lookup ti consentono di importare dati da record in altri progetti, creando visibilità unidirezionale. Mentre i [campi personalizzati Reference](/it/docs/custom-fields/reference) stabiliscono connessioni tra progetti, i Lookup utilizzano quelle connessioni per mostrare dati selezionati.

I campi personalizzati Lookup sono sempre di sola lettura e sono collegati a un campo personalizzato Reference specifico. Quando un utente seleziona uno o più record utilizzando un campo personalizzato Reference, il campo personalizzato Lookup mostrerà i dati di quei record.

I Lookup possono mostrare dati come:

- **Creato il**: Quando il record referenziato è stato creato.
- **Aggiornato il**: L'ultima data e ora in cui il record referenziato è stato aggiornato in qualsiasi modo.
- **Data di scadenza**: La data/ora di inizio/scadenza del record di riferimento.
- **Descrizione**: Il campo descrizione del record referenziato.
- **Lista**: Mostra la lista a cui appartiene il record referenziato.
- **Tag**: Qualsiasi tag con cui è etichettato il record referenziato.
- **Assegnatario**: Qualsiasi assegnatario del record referenziato.
- **Qualsiasi Campo Personalizzato**: Qualsiasi campo personalizzato supportato dal record referenziato.

I campi Lookup hanno molte opzioni 

## Lookup di un Lookup

Puoi creare campi personalizzati per archiviare e referenziare dati attraverso più progetti diversi. Questi campi personalizzati sono chiamati campi "Lookup".

Consideriamo tre progetti:

- **Progetto A**
- **Progetto B**
- **Progetto C**

Supponiamo che il Progetto B contenga alcune informazioni o risorse critiche che sono rilevanti sia per il Progetto A che per il Progetto C.

Invece di duplicare le informazioni dal Progetto B nel Progetto A e nel Progetto C, segui questi passaggi:

- Nel Progetto A, crea un campo personalizzato "Reference" per stabilire una connessione con il record rilevante dal Progetto B.

  - Questo campo personalizzato Reference ti consente di selezionare e collegare al record specifico nel Progetto B che vuoi referenziare e consultare.

- Una volta che il campo Reference è configurato nel Progetto A, crea un campo personalizzato "Lookup" nel Progetto C.

  - Questo campo personalizzato Lookup sarà collegato al campo personalizzato Reference nel Progetto A.

  - Quando selezioni record utilizzando il campo personalizzato Reference nel Progetto A, il campo personalizzato Lookup nel Progetto C mostrerà automaticamente i dati rilevanti da quei record referenziati nel Progetto B.

## Vantaggi dell'utilizzo di Lookup di un Lookup:

- Non hai bisogno di collegare direttamente il Progetto C al Progetto B.
- Il Progetto C può sfruttare la connessione stabilita tra il Progetto A e il Progetto B attraverso i campi Reference e Lookup.
- Qualsiasi modifica o aggiornamento apportato ai record rilevanti nel Progetto B si rifletterà automaticamente nei dati del campo personalizzato Lookup visualizzati nel Progetto C.
- Non è necessario aggiornare manualmente o duplicare le informazioni tra i progetti.
