## Aperçu

Les images de couverture vous permettent d'ajouter un contexte visuel à vos enregistrements en affichant des aperçus d'images directement sur les cartes d'enregistrement. Cette fonctionnalité extrait automatiquement les images de diverses sources au sein de vos enregistrements et les affiche sous forme de miniatures, facilitant l'identification et l'organisation du contenu visuel en un coup d'œil.

Que vous gériez des projets de design, des calendriers de contenu, des catalogues de produits ou tout flux de travail impliquant des éléments visuels, les images de couverture vous aident à parcourir rapidement les enregistrements et à trouver ce que vous cherchez. La fonctionnalité inclut un suivi des progrès en temps réel et des options d'affichage flexibles pour correspondre à vos besoins spécifiques de flux de travail.

## Fonctionnalités principales

- **Extraction automatique d'images** : Blue trouve et affiche automatiquement les images de votre source choisie
- **Sources d'images multiples** : Récupérez des images depuis les descriptions d'enregistrement, les commentaires ou les champs de fichiers personnalisés
- **Options d'ajustement d'affichage** : Choisissez comment les images apparaissent avec Couverture ou Contenu.
- **Sélection intelligente** : Configurez l'utilisation de la première ou dernière image de votre source
- **Mises à jour en temps réel** : Les images de couverture se mettent à jour automatiquement lorsque le contenu des enregistrements change
- **Suivi des progrès en direct** : Surveillez le traitement en lot avec des abonnements de progrès en temps réel

## Comment ça fonctionne

### Activation des images de couverture

Les images de couverture sont configurées au niveau du projet par les administrateurs de projet :

1. Ouvrez votre projet et naviguez vers **Paramètres du projet**
2. Sélectionnez **Paramètres d'enregistrement**
3. Trouvez la section **Options d'image de couverture**
4. Activez **Activer les images de couverture** pour activer la fonctionnalité

### Options de configuration

#### Source d'image
Choisissez où Blue doit chercher les images :
- **Description** : Extrait les images intégrées dans les descriptions d'enregistrement (par défaut)
- **Commentaires** : Récupère la première/dernière image des commentaires d'enregistrement
- **Champ personnalisé** : Utilise les fichiers d'un champ personnalisé de type fichier spécifique (prend en charge tout type de fichier)

#### Sélection d'image
Lorsque plusieurs images sont disponibles, choisissez laquelle afficher :
- **Première image** : Utilise la première image trouvée (par défaut)
- **Dernière image** : Utilise l'image la plus récente

#### Ajustement d'affichage
Contrôlez comment les images apparaissent dans la zone d'aperçu :
- **Couverture** : L'image remplit toute la zone d'aperçu, peut rogner les bords pour maintenir le rapport d'aspect (par défaut)
- **Contenu** : Affiche l'image complète dans la zone d'aperçu, peut avoir de l'espace vide

## Cas d'usage

**Projets de design** - Affichez les maquettes et itérations de design directement sur les cartes de tâches, facilitant la visualisation de l'état actuel de chaque tâche de design sans ouvrir d'enregistrements individuels.

**Planification de contenu** - Pour les calendriers de contenu et les flux de travail éditoriaux, affichez les images vedettes ou miniatures pour visualiser votre pipeline de contenu en un coup d'œil.

**Gestion de produit** - Affichez les photos de produits sur les enregistrements d'inventaire ou les demandes de fonctionnalités, aidant les équipes à identifier rapidement les articles lors des discussions.

**Immobilier** - Affichez les photos de propriété sur les enregistrements d'annonces, permettant aux agents de parcourir rapidement les propriétés visuellement.

**Planification d'événements** - Affichez les photos de lieux, concepts de design ou dispositions d'événements sur les tâches de planification pour une meilleure organisation visuelle.

## Configuration des images de couverture

### Étape 1 : Choisissez votre source d'image

Considérez où les images existent naturellement dans votre flux de travail :
- Si les images font partie des descriptions de tâches ou des briefs, utilisez **Description**
- Si les images sont ajoutées par la collaboration d'équipe, utilisez **Commentaires**
- Si vous avez un champ dédié pour les images, utilisez **Champ personnalisé**

### Étape 2 : Configurez les paramètres d'affichage

1. Sélectionnez votre **Ajustement d'image** préféré selon vos types d'images :
   - Utilisez **Couverture** pour les photos et designs où le rognage est acceptable
   - Utilisez **Contenu** pour les diagrammes ou captures d'écran où vous devez tout voir

2. Choisissez **Sélection d'image** selon votre flux de travail :
   - Utilisez **Première** pour les flux de travail où l'image principale est ajoutée initialement
   - Utilisez **Dernière** pour les flux de travail où l'image la plus récente est la plus pertinente

### Étape 3 : Traitement des enregistrements existants

Lorsque vous activez les images de couverture pour la première fois ou changez la configuration, Blue traite automatiquement vos enregistrements :
1. Les mises à jour de progrès en temps réel affichent le statut de traitement
2. Les enregistrements sont traités par lots (50 enregistrements par lot) pour des performances optimales
3. Vous pouvez continuer à travailler pendant que le traitement se déroule en arrière-plan
4. Les listes se rafraîchiront automatiquement lorsque le traitement sera terminé
5. Le traitement se déclenche automatiquement lorsque les changements de configuration sont sauvegardés

## Détails techniques

### Formats pris en charge
- Lors de l'utilisation des sources Description ou Commentaires :
  - Images JPEG/JPG
  - Images PNG (incluant la transparence)
  - Images GIF (affichage statique)
- Lors de l'utilisation de la source Champ personnalisé :
  - Tout type de fichier peut être affiché (images, PDF, documents)
  - Les fichiers non-images affichent des icônes de type de fichier appropriées
- Tous les fichiers doivent être téléchargés vers Blue (URLs externes non prises en charge)

### Limitations
- Maximum 6 000 enregistrements par projet pour la génération de couvertures en lot
- Seuls les champs personnalisés de type fichier peuvent être utilisés comme sources
- Les images de couverture sont en lecture seule et ne peuvent pas être modifiées directement

### Mises à jour automatiques
Les couvertures se régénèrent automatiquement lorsque :
- La description d'enregistrement est modifiée (si utilisation de la source Description)
- De nouveaux commentaires avec images sont ajoutés (si utilisation de la source Commentaires)
- Les fichiers dans les champs personnalisés sont mis à jour (si utilisation de la source Champ personnalisé)
- Les paramètres de configuration de couverture sont modifiés
- L'option d'ajustement d'affichage est modifiée

### Accès API
Les images de couverture sont entièrement intégrées avec l'API GraphQL :
- Configurez via la mutation `setTodoCoverConfig`
- Accédez aux données de couverture via le champ `cover` sur le type Todo
- Surveillez les progrès avec l'abonnement `subscribeToCoverGenerationProgress`
- URLs de miniatures générées automatiquement à `/uploads/{uid}/thumbnail/cover`
