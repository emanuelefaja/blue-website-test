Le champ ID Unique génère automatiquement un identifiant unique pour chaque enregistrement. Blue propose deux types de systèmes d'identification unique :

## 1. ID Alphanumériques Aléatoires (Par défaut)

Le système d'ID Unique standard génère un identifiant alphanumérique unique pour chaque enregistrement. Cela apporte une valeur immense en créant des clés persistantes qui identifient de manière cohérente chaque enregistrement. Même si un enregistrement est supprimé, son ID ne sera jamais dupliqué.

## 2. ID Séquentiels

Pour les projets nécessitant des identifiants numérotés lisibles par l'homme, Blue prend également en charge les ID séquentiels. Lorsqu'il est activé, ce système génère des numéros incrémentaux (par exemple, 1, 2, 3, ou avec un remplissage de zéros comme 001, 002, 003) qui peuvent être personnalisés avec des préfixes (par exemple, TASK-001, TICKET-002).

## Cas d'Usage

Les deux systèmes d'ID permettent des capacités puissantes :

**Avantages Généraux :**
- Associer des systèmes externes aux enregistrements via des codes uniques
- Créer des pistes d'audit en suivant les références d'ID
- Rechercher facilement des enregistrements dans la plateforme via l'ID, même si le nom de l'enregistrement change

**Avantages Spécifiques aux ID Séquentiels :**
- Références de tâches lisibles par l'homme pour la communication d'équipe
- Schémas de numérotation cohérents pour les flux de travail (par exemple, tickets de support, demandes de fonctionnalités)
- Intégration facile avec des systèmes externes qui attendent des identifiants numériques
- Génération automatique de titres avec des préfixes séquentiels

![UniqueID 1](/docs/UniqueID_1.png "UniqueID 1")

Les deux systèmes d'ID établissent essentiellement des "numéros de sécurité sociale" immuables pour chaque enregistrement. Cette permanence et cette spécificité permettent des intégrations, des analyses et des capacités d'identification puissantes.

![UniqueID 2](/docs/UniqueID_2.png "UniqueID 2")

## Configuration des ID Séquentiels

Lors de la création ou de la modification d'un champ personnalisé ID Unique, vous pouvez activer la numérotation séquentielle avec ces options :

- **Utiliser un ID Unique Séquentiel** : Basculer pour activer la numérotation séquentielle au lieu des ID alphanumériques aléatoires
- **Chiffres de Séquence** : Nombre de chiffres pour le remplissage de zéros (par exemple, 3 chiffres = 001, 002, 003)
- **Numéro de Démarrage de Séquence** : Le numéro à partir duquel commencer le comptage (par défaut : 1)
- **Préfixe** : Préfixe de texte optionnel pour les titres générés (par exemple, "TASK-" crée TASK-001, TASK-002)

**Total d'ID Uniques Disponibles** <br/><br/>
Il y a 36 possibilités pour chacun des 25 caractères dans un ID Unique Blue : <br/>
- 26 lettres majuscules (A-Z) <br/>
- 10 chiffres (0-9) <br/>

Cela signifie qu'il y a un total de 8,08×10^35 possibilités ou, pour être plus précis : 808 281 277 464 764 060 643 139 600 456 536 293 376 <br/><br/>
Pour mettre cela en comparaison, pour chaque grain de sable sur Terre (7,5*10^18), il y a des dizaines de milliers de milliards (1,08×10^17) d'ID Uniques !

**Détails Techniques des ID Séquentiels** <br/><br/>
Les ID séquentiels sont limités au projet, ce qui signifie que chaque projet maintient son propre compteur de séquence. Lorsque les enregistrements sont copiés ou déplacés entre projets, le système gère intelligemment l'attribution de séquence basée sur les configurations de champs personnalisés correspondantes. La génération de séquence est atomique et thread-safe pour éviter les numéros en double dans les environnements concurrents.
