Les champs personnalisés de consultation vous permettent d'importer des données d'enregistrements d'autres projets, créant une visibilité unidirectionnelle. Alors que les [champs personnalisés de référence](/fr/docs/custom-fields/reference) établissent des connexions entre projets, les consultations utilisent ces connexions pour afficher des données sélectionnées.

Les champs personnalisés de consultation sont toujours en lecture seule et sont connectés à un champ personnalisé de référence spécifique. Lorsqu'un utilisateur sélectionne un ou plusieurs enregistrements à l'aide d'un champ personnalisé de référence, le champ personnalisé de consultation affichera les données de ces enregistrements.

Les consultations peuvent afficher des données telles que :

- **Créé le** : Quand l'enregistrement référencé a été créé.
- **Mis à jour le** : La dernière date et heure où l'enregistrement référencé a été mis à jour de quelque façon que ce soit.
- **Date d'échéance** : La date et l'heure de début/échéance de l'enregistrement de référence.
- **Description** : Le champ description de l'enregistrement référencé.
- **Liste** : Affiche la liste à laquelle appartient l'enregistrement référencé.
- **Étiquette** : Toutes les étiquettes avec lesquelles l'enregistrement référencé est étiqueté.
- **Assigné** : Tous les assignés à l'enregistrement référencé.
- **Tout champ personnalisé** : Tout champ personnalisé pris en charge de l'enregistrement référencé.

## Consulter une consultation

Vous pouvez créer des champs personnalisés pour stocker et référencer des données à travers plusieurs projets différents. Ces champs personnalisés sont appelés champs de "Consultation".

Considérons trois projets :

- **Projet A**
- **Projet B**
- **Projet C**

Supposons que le Projet B contient des informations critiques ou des ressources qui sont pertinentes pour le Projet A et le Projet C.

Au lieu de dupliquer les informations du Projet B dans le Projet A et le Projet C, suivez ces étapes :

- Dans le Projet A, créez un champ personnalisé "Référence" pour établir une connexion avec l'enregistrement pertinent du Projet B.

  - Ce champ personnalisé de référence vous permet de sélectionner et de lier l'enregistrement spécifique du Projet B que vous voulez référencer et consulter.

- Une fois que le champ de référence est configuré dans le Projet A, créez un champ personnalisé "Consultation" dans le Projet C.

  - Ce champ personnalisé de consultation sera connecté au champ personnalisé de référence dans le Projet A.

  - Lorsque vous sélectionnez des enregistrements en utilisant le champ personnalisé de référence dans le Projet A, le champ personnalisé de consultation dans le Projet C affichera automatiquement les données pertinentes de ces enregistrements référencés dans le Projet B.

## Avantages de l'utilisation de Consulter une consultation :

- Vous n'avez pas besoin de lier directement le Projet C au Projet B.
- Le Projet C peut tirer parti de la connexion établie entre le Projet A et le Projet B grâce aux champs de référence et de consultation.
- Tous les changements ou mises à jour apportés aux enregistrements pertinents dans le Projet B se refléteront automatiquement dans les données du champ personnalisé de consultation affichées dans le Projet C.
- Aucun besoin de mises à jour manuelles ou de duplication d'informations entre projets.
