Поле Уникальный ID автоматически генерирует уникальный идентификатор для каждой записи. Blue предлагает два типа систем уникальной идентификации:

## 1. Случайные буквенно-цифровые ID (по умолчанию)

Стандартная система Уникальный ID генерирует уникальный буквенно-цифровой идентификатор для каждой записи. Это приносит огромную пользу, создавая постоянные ключи, которые последовательно идентифицируют каждую запись. Даже если запись удалена, её ID никогда не будет дублирован.

## 2. Последовательные ID

Для проектов, требующих читаемых человеком, пронумерованных идентификаторов, Blue также поддерживает последовательные ID. При включении эта система генерирует возрастающие номера (например, 1, 2, 3, или с дополнением нулями как 001, 002, 003), которые можно настроить с префиксами (например, TASK-001, TICKET-002).

## Случаи использования

Обе системы ID обеспечивают мощные возможности:

**Общие преимущества:**
- Привязка внешних систем к записям через уникальные коды
- Создание аудиторских следов путём отслеживания ссылок на ID
- Легкий поиск записей в платформе по ID, даже если название записи изменяется

**Специфические преимущества последовательных ID:**
- Читаемые человеком ссылки на задачи для командного общения
- Последовательные схемы нумерации для рабочих процессов (например, тикеты поддержки, запросы функций)
- Легкая интеграция с внешними системами, которые ожидают числовые идентификаторы
- Автоматическое создание заголовков с последовательными префиксами

![UniqueID 1](/docs/UniqueID_1.png "UniqueID 1")

Обе системы ID по сути устанавливают неизменяемые "социальные номера" для каждой записи. Эта постоянность и специфичность обеспечивает мощные возможности интеграции, аналитики и идентификации.

![UniqueID 2](/docs/UniqueID_2.png "UniqueID 2")

## Настройка последовательных ID

При создании или редактировании пользовательского поля Уникальный ID вы можете включить последовательную нумерацию с этими опциями:

- **Использовать последовательный уникальный ID**: Переключатель для включения последовательной нумерации вместо случайных буквенно-цифровых ID
- **Цифры последовательности**: Количество цифр для дополнения нулями (например, 3 цифры = 001, 002, 003)
- **Начальный номер последовательности**: Номер, с которого начинать отсчёт (по умолчанию: 1)
- **Префикс**: Необязательный текстовый префикс для генерируемых заголовков (например, "TASK-" создаёт TASK-001, TASK-002)

**Всего доступных уникальных ID** <br/><br/>
Есть 36 возможностей для каждого из 25 символов в уникальном ID Blue: <br/>
- 26 заглавных букв (A-Z) <br/>
- 10 цифр (0-9) <br/>

Это означает, что всего есть 8,08×10^35 возможностей или, точнее: 808 281 277 464 764 060 643 139 600 456 536 293 376 <br/><br/>
Для сравнения, на каждую песчинку на Земле (7,5*10^18) приходятся десятки квадриллионов (1,08×10&17) уникальных ID!
::

Администраторы проекта могут настроить пользовательский префикс для всех случайных буквенно-цифровых уникальных ID.

**Технические детали последовательных ID** <br/><br/>
Последовательные ID ограничены проектом, то есть каждый проект поддерживает свой собственный счётчик последовательности. Когда записи копируются или перемещаются между проектами, система интеллектуально обрабатывает назначение последовательности на основе соответствующих конфигураций пользовательских полей. Генерация последовательности является атомарной и потокобезопасной для предотвращения дублирования номеров в параллельных средах.
