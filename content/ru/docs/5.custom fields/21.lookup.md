Пользовательские поля поиска позволяют импортировать данные из записей в других проектах, создавая одностороннюю видимость. Если [пользовательские поля ссылок](/ru/docs/custom-fields/reference) устанавливают связи между проектами, то поля поиска используют эти связи для отображения выбранных данных.

Пользовательские поля поиска всегда доступны только для чтения и связаны с определенным пользовательским полем ссылки. Когда пользователь выбирает одну или несколько записей с помощью пользовательского поля ссылки, поле поиска отобразит данные из этих записей.

Поля поиска могут отображать данные, такие как:

- **Дата создания**: Когда была создана связанная запись.
- **Дата обновления**: Последняя дата и время, когда связанная запись была изменена каким-либо образом.
- **Срок выполнения**: Дата и время начала/срока выполнения связанной записи.
- **Описание**: Поле описания связанной записи.
- **Список**: Показывает список, к которому принадлежит связанная запись.
- **Тег**: Любые теги, которыми помечена связанная запись.
- **Исполнитель**: Любые исполнители, назначенные на связанную запись.
- **Любое пользовательское поле**: Любое поддерживаемое пользовательское поле из связанной записи.

## Поиск поиска

Вы можете создавать пользовательские поля для хранения и ссылки на данные в нескольких различных проектах. Эти пользовательские поля называются полями "Поиска".

Рассмотрим три проекта:

- **Проект A**
- **Проект B**
- **Проект C**

Предположим, что проект B содержит важную информацию или ресурсы, которые имеют отношение как к проекту A, так и к проекту C.

Вместо дублирования информации из проекта B в проекты A и C, выполните следующие шаги:

- В проекте A создайте пользовательское поле "Ссылка" для установления связи с соответствующей записью из проекта B.

  - Это пользовательское поле ссылки позволяет выбрать и связать конкретную запись в проекте B, на которую вы хотите ссылаться и которую хотите найти.

- После настройки поля ссылки в проекте A создайте пользовательское поле "Поиск" в проекте C.

  - Это пользовательское поле поиска будет связано с пользовательским полем ссылки в проекте A.

  - Когда вы выбираете записи с помощью пользовательского поля ссылки в проекте A, пользовательское поле поиска в проекте C автоматически отобразит соответствующие данные из этих связанных записей в проекте B.

## Преимущества использования поиска поиска:

- Вам не нужно напрямую связывать проект C с проектом B.
- Проект C может использовать связь, установленную между проектом A и проектом B через поля ссылки и поиска.
- Любые изменения или обновления, внесенные в соответствующие записи в проекте B, автоматически отразятся в данных пользовательского поля поиска, отображаемых в проекте C.
- Нет необходимости в ручных обновлениях или дублировании информации между проектами.
