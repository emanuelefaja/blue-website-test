---
title: Архивирование проектов
description: Заморозьте проекты после завершения для соответствия требованиям.
---

Во многих организациях сохранение данных завершенных проектов имеет решающее значение для целей аудита или для пересмотра проекта в будущем. Однако накопление сотен завершенных проектов может загромождать интерфейс Blue, затрудняя навигацию и управление текущими проектами.

Именно поэтому Blue предлагает возможность для [Администраторов проектов](/docs/user-management/roles/project-administrator) архивировать проекты после их завершения или когда они больше не требуются.

![Archiving Project 1](/docs/ArchivingProject_1.png "Archiving Project 1")

Легко архивируйте любой проект

Когда проект архивируется, происходит несколько вещей.

- **Перемещение в 'Архивированные проекты'**: Проект перемещается из основной области 'Проекты' на боковой панели в специальный раздел 'Архивированные проекты'. Это помогает поддерживать чистый и организованный основной интерфейс.
- **Видимый статус архивации**: В верхней панели рядом с названием проекта появляется метка 'Архивирован', ясно указывающая на его статус.
- **Доступ только для чтения**: Архивированные проекты становятся доступными только для чтения. Они по сути 'заморожены во времени', что означает, что все пользователи могут просматривать проект и его содержимое, но никакие изменения или дополнения не могут быть внесены. Это обеспечивает целостность и неизменность данных проекта такими, какими они были на момент архивации.
- **Изменения доступа к API**: Хотя проект все еще может быть доступен через API Blue для операций чтения, любые операции записи (попытки изменить или добавить данные) не будут успешными. Это поддерживает статус только для чтения проекта даже через взаимодействия с API.
