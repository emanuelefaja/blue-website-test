Pole Unikalny ID automatycznie generuje unikalny identyfikator dla każdego rekordu. Blue oferuje dwa typy systemów unikalnej identyfikacji:

## 1. Losowe ID alfanumeryczne (Domyślne)

Standardowy system Unikalnego ID generuje unikalny identyfikator alfanumeryczny dla każdego rekordu. Przynosi to ogromną wartość poprzez tworzenie trwałych kluczy, które konsekwentnie identyfikują każdy rekord. Nawet jeśli rekord zostanie usunięty, jego ID nigdy nie zostanie powielone.

## 2. ID sekwencyjne

Dla projektów wymagających czytelnych dla człowieka, numerowanych identyfikatorów, Blue obsługuje również ID sekwencyjne. Po włączeniu ten system generuje liczby przyrostowe (np. 1, 2, 3, lub z wypełnieniem zerami jak 001, 002, 003), które można dostosować za pomocą prefiksów (np. TASK-001, TICKET-002).

## Przypadki użycia

Oba systemy ID umożliwiają potężne funkcjonalności:

**Ogólne korzyści:**
- Kojarzenie systemów zewnętrznych z rekordami za pomocą unikalnych kodów
- Tworzenie ścieżek audytu poprzez śledzenie referencji ID
- Łatwe wyszukiwanie rekordów w platformie za pomocą ID, nawet jeśli nazwa rekordu się zmieni

**Korzyści specyficzne dla ID sekwencyjnych:**
- Czytelne dla człowieka odniesienia do zadań w komunikacji zespołowej
- Spójne schematy numeracji dla przepływów pracy (np. zgłoszenia wsparcia, prośby o funkcje)
- Łatwa integracja z systemami zewnętrznymi, które oczekują identyfikatorów numerycznych
- Automatyczne generowanie tytułów z prefiksami sekwencyjnymi

Oba systemy ID zasadniczo ustanawiają niezmienne "numery PESEL" dla każdego rekordu. Ta trwałość i specyficzność umożliwia potężne integracje, analizy i możliwości identyfikacji.

## Konfiguracja ID sekwencyjnego

Podczas tworzenia lub edytowania pola niestandardowego Unikalny ID, możesz włączyć numerację sekwencyjną z następującymi opcjami:

- **Użyj sekwencyjnego unikalnego ID**: Przełącznik do włączenia numeracji sekwencyjnej zamiast losowych ID alfanumerycznych
- **Cyfry sekwencji**: Liczba cyfr do wypełnienia zerami (np. 3 cyfry = 001, 002, 003)
- **Numer początkowy sekwencji**: Liczba, od której zacząć liczenie (domyślnie: 1)
- **Prefiks**: Opcjonalny prefiks tekstowy dla generowanych tytułów (np. "TASK-" tworzy TASK-001, TASK-002)

**Całkowita liczba dostępnych unikalnych ID** <br/><br/>
Istnieje 36 możliwości dla każdego z 25 znaków w Blue Unikalnym ID: <br/>
- 26 wielkich liter (A-Z) <br/>
- 10 cyfr (0-9) <br/>

Oznacza to, że istnieje łącznie 8,08×10^35 możliwości lub, żeby być bardziej precyzyjnym: 808,281,277,464,764,060,643,139,600,456,536,293,376 <br/><br/>
Dla porównania, na każde ziarnko piasku na Ziemi (7,5*10^18), przypadają dziesiątki kwadrylionów (1,08×10&17) unikalnych ID!
::

## Konfiguracja losowego ID alfanumerycznego

Administratorzy projektów mogą skonfigurować niestandardowy prefiks dla wszystkich losowych alfanumerycznych unikalnych ID.

**Szczegóły techniczne ID sekwencyjnych** <br/><br/>
ID sekwencyjne są ograniczone do zakresu projektu, co oznacza, że każdy projekt utrzymuje własny licznik sekwencji. Gdy rekordy są kopiowane lub przenoszone między projektami, system inteligentnie obsługuje przypisywanie sekwencji na podstawie pasujących konfiguracji pól niestandardowych. Generowanie sekwencji jest atomowe i bezpieczne wątkowo, aby zapobiec duplikacji numerów w środowiskach współbieżnych.
::
