Tags sind eine nützliche Möglichkeit, Ihre Arbeit zu organisieren. Sie können sie verwenden, um verschiedene Aufgabentypen, Dringlichkeit oder jede andere Kategorie zu kennzeichnen, die zu Ihrem Projekt passt. Jeder Tag kann farblich kodiert werden, um eine einfache Identifizierung zu ermöglichen und Ihrem Arbeitsbereich etwas Farbe hinzuzufügen.

<youtube url="https://www.youtube.com/watch?v=nlyY3cUUpkQ" />


## Tags verwenden

- **Tags hinzufügen**: Um einen Tag zu erstellen, betrachten Sie die Aspekte Ihrer Aufgaben, die eine Kategorisierung benötigen – sei es die Art der Aufgabe, wie 'Bug', 'Back-End' oder 'UI', oder deren Priorität. Sobald Sie Ihre Kategorien identifiziert haben, können Sie einfach Tags erstellen und ihnen eindeutige Farben für sofortige Erkennbarkeit zuweisen.
- **Filtern mit Tags**: Die Filterfunktionen von Blue machen die Verwaltung Ihrer Arbeitsbelastung intuitiver. Sie können Tags über das Dropdown-Filter-Symbol filtern oder direkt in der Board-Ansicht, indem Sie auf den gewünschten Tag klicken.

Projektübergreifende Tag-Nutzung <br/>
Für organisationsweite Ansichten wie Dashboards und Übersichten wird das Taggen besonders wichtig, da Blue automatisch Tags mit demselben Namen zusammenführt. Sie müssen nur nach dem "Priorität 1"-Tag filtern, was nur Datensätze aus allen Projekten mit diesem Tag anzeigt.
