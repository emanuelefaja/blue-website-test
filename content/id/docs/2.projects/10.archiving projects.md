---
title: Mengarsipkan proyek
description: Bekukan proyek setelah selesai untuk kepatuhan.
---

Di banyak organisasi, menyimpan data dari proyek yang telah selesai sangat penting untuk tujuan audit atau untuk meninjau kembali proyek di masa depan. Namun, mengakumulasi ratusan proyek yang telah selesai dapat membuat antarmuka Blue menjadi berantakan, menantang navigasi dan manajemen proyek saat ini.

Inilah mengapa Blue menawarkan kemampuan bagi [Administrator Proyek](/docs/user-management/roles/project-administrator) untuk mengarsipkan proyek setelah selesai atau tidak diperlukan lagi.

![Archiving Project 1](/docs/ArchivingProject_1.png "Archiving Project 1")

Arsipkan proyek apa pun dengan mudah

Ketika sebuah proyek diarsipkan, beberapa hal terjadi.

- **Pemindahan ke 'Proyek Terarsip'**: Proyek dipindahkan dari area utama 'Proyek' di sidebar ke bagian khusus 'Proyek Terarsip'. Ini membantu mempertahankan antarmuka utama yang bersih dan terorganisir.
- **Status Terarsip yang Terlihat**: Di bilah atas, label 'Terarsip' muncul di sebelah nama proyek, dengan jelas menunjukkan statusnya.
- **Akses Hanya-Baca**: Proyek yang diarsipkan menjadi hanya-baca. Mereka pada dasarnya 'dibekukan dalam waktu', artinya semua pengguna dapat melihat proyek dan isinya, tetapi tidak ada modifikasi atau penambahan yang dapat dilakukan. Ini memastikan integritas dan ketidakberubahan data proyek seperti saat pengarsipan.
- **Perubahan Akses API**: Meskipun proyek masih dapat diakses melalui API Blue untuk operasi baca, operasi tulis apa pun (upaya untuk memodifikasi atau menambahkan data) tidak akan berhasil. Ini mempertahankan status hanya-baca proyek bahkan melalui interaksi API.
