Field ID Unik secara otomatis menghasilkan pengidentifikasi unik untuk setiap record. Blue menawarkan dua jenis sistem identifikasi unik:

## 1. ID Alfanumerik Acak (Default)

Sistem ID Unik standar menghasilkan pengidentifikasi alfanumerik unik untuk setiap record. Ini memberikan nilai yang sangat besar dengan menciptakan kunci persisten yang secara konsisten mengidentifikasi setiap record. Bahkan jika record dihapus, ID-nya tidak akan pernah diduplikasi.

## 2. ID Berurutan

Untuk proyek yang memerlukan pengidentifikasi bernomor yang dapat dibaca manusia, Blue juga mendukung ID berurutan. Ketika diaktifkan, sistem ini menghasilkan angka bertambah (misalnya, 1, 2, 3, atau dengan zero-padding seperti 001, 002, 003) yang dapat disesuaikan dengan awalan (misalnya, TASK-001, TICKET-002).

## Kasus Penggunaan

Kedua sistem ID memungkinkan kemampuan yang kuat:

**Manfaat Umum:**
- Mengasosiasikan sistem eksternal ke record melalui kode unik
- Membuat audit trail dengan melacak referensi ID
- Mencari record dengan mudah di platform melalui ID, bahkan jika nama record berubah

**Manfaat Khusus ID Berurutan:**
- Referensi tugas yang dapat dibaca manusia untuk komunikasi tim
- Skema penomoran yang konsisten untuk workflow (misalnya, tiket dukungan, permintaan fitur)
- Integrasi mudah dengan sistem eksternal yang mengharapkan pengidentifikasi numerik
- Pembuatan judul otomatis dengan awalan berurutan

![UniqueID 1](/docs/UniqueID_1.png "UniqueID 1")

Kedua sistem ID pada dasarnya menetapkan "nomor jaminan sosial" yang tidak dapat diubah untuk setiap record. Permanensi dan spesifisitas ini memungkinkan integrasi, analitik, dan kemampuan identifikasi yang kuat.

![UniqueID 2](/docs/UniqueID_2.png "UniqueID 2")

## Konfigurasi ID Berurutan

Saat membuat atau mengedit field kustom ID Unik, Anda dapat mengaktifkan penomoran berurutan dengan opsi-opsi berikut:

- **Gunakan ID Unik Berurutan**: Toggle untuk mengaktifkan penomoran berurutan alih-alih ID alfanumerik acak
- **Digit Urutan**: Jumlah digit untuk zero-padding (misalnya, 3 digit = 001, 002, 003)
- **Nomor Awal Urutan**: Nomor untuk memulai penghitungan (default: 1)
- **Awalan**: Awalan teks opsional untuk judul yang dihasilkan (misalnya, "TASK-" membuat TASK-001, TASK-002)

**Total ID Unik yang Tersedia** <br/><br/>
Ada 36 kemungkinan untuk setiap 25 karakter dalam ID Unik Blue: <br/>
- 26 huruf besar (A-Z) <br/>
- 10 digit (0-9) <br/>

Ini berarti bahwa ada total 8,08×10^35 kemungkinan atau, lebih akurat: 808,281,277,464,764,060,643,139,600,456,536,293,376 <br/><br/>
Untuk menempatkan ini dalam perbandingan, untuk setiap butir pasir di Bumi (7,5*10^18), ada puluhan kuadriliun (1,08×10&17) ID Unik!

**Detail Teknis ID Berurutan** <br/><br/>
ID berurutan memiliki cakupan proyek, yang berarti setiap proyek mempertahankan penghitung urutan sendiri. Ketika record disalin atau dipindahkan antar proyek, sistem secara cerdas menangani penugasan urutan berdasarkan konfigurasi field kustom yang cocok. Pembuatan urutan bersifat atomik dan thread-safe untuk mencegah nomor duplikat dalam lingkungan konkuren.
