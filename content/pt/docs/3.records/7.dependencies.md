title: Dependências

Ao configurar dependências de registros, você pode garantir que as tarefas sejam concluídas na ordem correta de operações.

Existem dois tipos de dependências:

1. **Bloqueando —** O registro bloqueia outros registros de iniciar ou concluir até que seja finalizado. Outros registros dependem deste ser concluído primeiro.
2. **Bloqueado Por —** O registro é bloqueado por outro registro que deve ser concluído primeiro antes que este possa iniciar ou terminar. Este registro depende de outro ser concluído antes de poder prosseguir.

![Dependencies 1](/docs/Dependencies_1.gif "Dependencies 1")

## Criando Dependências

Para configurar uma dependência:
1. Vá para o registro que deve ser concluído primeiro.
2. Clique em Dependências abaixo da seção Tags.
3. Escolha "Bloqueando" para fazer este registro bloquear outro.
4. Selecione o registro que você deseja que este bloqueie no menu suspenso.
5. Clique em Adicionar.

O registro bloqueado agora mostrará "Bloqueado por" e será impedido de ser concluído até que o registro bloqueador seja finalizado.

## Concluindo Registros Bloqueados

Se um usuário tentar marcar um registro bloqueado como concluído antes que suas dependências sejam satisfeitas, um popup de aviso aparecerá, alertando-o sobre a dependência bloqueadora. Isso evita que registros sejam concluídos fora de ordem por engano.

![Dependencies 2](/docs/Dependencies_2.gif "Dependencies 2")

## Removendo Dependências

Para remover uma dependência, vá para qualquer um dos registros vinculados. Em Dependências, clique no menu suspenso e selecione "Remover". Isso excluirá o vínculo entre os registros.

## Rastreamento de Dependências

Todas as dependências são registradas na seção Atividade dos registros, fornecendo transparência nas conexões do fluxo de trabalho. Isso permite rastrear a sequência de conclusão de registros interconectados.

As dependências permitem que as equipes modelem fluxos de trabalho e evitem a conclusão de registros fora de ordem ou em sequência inadequada. Conectar registros estabelece ordem e estrutura para projetos multifuncionais complexos.
