O campo ID Único gera automaticamente um identificador único para cada registro. O Blue oferece dois tipos de sistemas de identificação única:

## 1. IDs Alfanuméricos Aleatórios (Padrão)

O sistema padrão de ID Único gera um identificador alfanumérico único para cada registro. Isso traz imenso valor ao criar chaves persistentes que identificam consistentemente cada registro. Mesmo que um registro seja excluído, seu ID nunca será duplicado.

## 2. IDs Sequenciais

Para projetos que requerem identificadores numerados e legíveis por humanos, o Blue também suporta IDs sequenciais. Quando habilitado, este sistema gera números incrementais (ex.: 1, 2, 3, ou com zeros à esquerda como 001, 002, 003) que podem ser personalizados com prefixos (ex.: TASK-001, TICKET-002).

## Casos de Uso

Ambos os sistemas de ID habilitam capacidades poderosas:

**Benefícios Gerais:**
- Associar sistemas externos aos registros via códigos únicos
- Criar trilhas de auditoria rastreando referências de ID
- Pesquisar registros facilmente na plataforma via ID, mesmo que o nome do registro mude

**Benefícios Específicos do ID Sequencial:**
- Referências de tarefas legíveis por humanos para comunicação em equipe
- Esquemas de numeração consistentes para fluxos de trabalho (ex.: tickets de suporte, solicitações de recursos)
- Integração fácil com sistemas externos que esperam identificadores numéricos
- Geração automática de títulos com prefixos sequenciais

Ambos os sistemas de ID essencialmente estabelecem "números de CPF" imutáveis para cada registro. Esta permanência e especificidade possibilita integrações poderosas, análises e capacidades de identificação.

## Configuração de ID Sequencial

Ao criar ou editar um campo personalizado de ID Único, você pode habilitar numeração sequencial com estas opções:

- **Usar ID Único Sequencial**: Alternar para habilitar numeração sequencial em vez de IDs alfanuméricos aleatórios
- **Dígitos da Sequência**: Número de dígitos para preenchimento com zeros (ex.: 3 dígitos = 001, 002, 003)
- **Número Inicial da Sequência**: O número para começar a contar (padrão: 1)
- **Prefixo**: Prefixo de texto opcional para títulos gerados (ex.: "TASK-" cria TASK-001, TASK-002)

**Total de IDs Únicos Disponíveis** <br/><br/>
Existem 36 possibilidades para cada um dos 25 caracteres em um ID Único do Blue: <br/>
- 26 letras maiúsculas (A-Z) <br/>
- 10 dígitos (0-9) <br/>

Isso significa que há um total de 8,08×10^35 possibilidades ou, para ser mais preciso: 808.281.277.464.764.060.643.139.600.456.536.293.376 <br/><br/>
Para colocar isso em comparação, para cada grão de areia na Terra (7,5*10^18), existem dezenas de quatrilhões (1,08×10&17) de IDs Únicos!

**Detalhes Técnicos do ID Sequencial** <br/><br/>
IDs Sequenciais têm escopo de projeto, significando que cada projeto mantém seu próprio contador de sequência. Quando registros são copiados ou movidos entre projetos, o sistema inteligentemente gerencia a atribuição de sequência baseada em configurações de campos personalizados correspondentes. A geração de sequência é atômica e thread-safe para prevenir números duplicados em ambientes concorrentes.
