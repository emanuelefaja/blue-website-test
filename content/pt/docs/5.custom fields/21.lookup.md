Os campos personalizados Lookup permitem importar dados de registros em outros projetos, criando visibilidade unidirecional. Enquanto os [campos personalizados de Referência](/pt/docs/custom-fields/reference) estabelecem conexões entre projetos, os Lookups usam essas conexões para exibir dados selecionados.

Os campos personalizados Lookup são sempre somente leitura e estão conectados a um campo personalizado de Referência específico. Quando um usuário seleciona um ou mais registros usando um campo personalizado de Referência, o campo personalizado Lookup mostrará dados desses registros.

Os Lookups podem mostrar dados como:

- **Criado em**: Quando o registro referenciado foi criado.
- **Atualizado em**: A última data e hora em que o registro referenciado foi atualizado de qualquer forma.
- **Data de Vencimento**: A data/hora de início/vencimento do registro de referência.
- **Descrição**: O campo de descrição do registro referenciado.
- **Lista**: Mostra a lista à qual o registro referenciado pertence.
- **Tag**: Quaisquer tags com as quais o registro referenciado está marcado.
- **Responsável**: Quaisquer responsáveis pelo registro referenciado.
- **Qualquer Campo Personalizado**: Qualquer campo personalizado suportado do registro referenciado.

## Lookup de um Lookup

Você pode criar campos personalizados para armazenar e referenciar dados em vários projetos diferentes. Esses campos personalizados são chamados de campos "Lookup".

Vamos considerar três projetos:

- **Projeto A**
- **Projeto B**
- **Projeto C**

Suponha que o Projeto B contém algumas informações críticas ou recursos que são relevantes tanto para o Projeto A quanto para o Projeto C.

Em vez de duplicar as informações do Projeto B no Projeto A e no Projeto C, siga estes passos:

- No Projeto A, crie um campo personalizado de "Referência" para estabelecer uma conexão com o registro relevante do Projeto B.

  - Este campo personalizado de Referência permite que você selecione e vincule ao registro específico no Projeto B que você deseja referenciar e fazer lookup.

- Uma vez que o campo de Referência esteja configurado no Projeto A, crie um campo personalizado "Lookup" no Projeto C.

  - Este campo personalizado Lookup será conectado ao campo personalizado de Referência no Projeto A.

  - Quando você selecionar registros usando o campo personalizado de Referência no Projeto A, o campo personalizado Lookup no Projeto C automaticamente exibirá dados relevantes desses registros referenciados no Projeto B.

## Benefícios de usar Lookup de um Lookup:

- Você não precisa vincular diretamente o Projeto C ao Projeto B.
- O Projeto C pode aproveitar a conexão estabelecida entre o Projeto A e o Projeto B através dos campos de Referência e Lookup.
- Quaisquer alterações ou atualizações feitas nos registros relevantes no Projeto B serão automaticamente refletidas nos dados do campo personalizado Lookup exibidos no Projeto C.
- Não há necessidade de atualizações manuais ou duplicação de informações entre projetos.
